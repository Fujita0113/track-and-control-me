# 依頼: 「30日チャレンジ完了レポート」画面のUIモック

ゲーム解禁を人質に日次ルールを強制する個人用ローカルアプリの新機能。30日チャレンジの最終日に開くレポート画面を、静的HTML1枚（デスクトップ・日本語UI）でモックして。

構成は下のヘッダ＋4ブロックだけ。1カラム・余白広め・静かに。**指定以外の要素を足さないこと。**

## ヘッダ

目標名「メンタルを安定させる」/ 目的の一文 / 期間 2026-06-11〜07-10 / 達成 24/30。
合否・判定・スコアの言葉は使わない。「完走」とだけ。

## ① 達成カレンダー

30日×実践ごとの行。やった/やってないの2値。この目標の実践は3行:
振り返りを書く / 明日のタスク登録 / 作業4時間。

## ② 時間の推移（時間型の実践だけが持つブロック）

「作業4時間」の日々の実測と、その日の閾値。Day13 に閾値を 4h→3h に下げた記録があり、
理由の一言「課題週間。ゼロにはしない」を添える。恥じる見せ方をしない——下げて、続けた、という事実として。

## ③ Before / After

Day1 と Day30 の振り返り文面を左右に並置。

## ④ 毎日の日記

この目標について毎日書いた日記を読み返せる。30件を並べて表示せず、**読むのは常に1件**——
日付を選ぶとその日の本文が出る（①のカレンダーのマスをクリックでその日へ飛ぶ、でもよい）。
