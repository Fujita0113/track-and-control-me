# timeline-gap-recording Specification

## Purpose

未記録ギャップをタイムライン上の一級市民（ゴーストスロット）として可視化し、ワンクリックで記録できるようにする。「吸収する微小離席」と「記録すべき離席」の境界を単一の設定値 `away_min_seconds` に一元化する。

## Requirements

### Requirement: 未記録ギャップのゴーストスロット表示
タイムラインは、サーバーが返す未カバー区間（`gaps`）を「＋ 未記録 HH:MM–HH:MM（n分）」のゴーストスロット（破線アウトライン）としてレーンの実時間位置に描画する SHALL。サーバーは `away_min_seconds` 以上のギャップのみを返し、未満の区間はゴーストとして表示されない SHALL。

#### Scenario: 閾値以上のギャップがゴーストになる
- **WHEN** 12:42–13:05（23分、閾値10分以上）にどのブロックにも覆われない区間がある
- **THEN** その位置に「＋ 未記録 12:42–13:05（23分）」のゴーストスロットが表示される

#### Scenario: 閾値未満のギャップは表示されない
- **WHEN** 6分（閾値10分未満）の未カバー区間がある
- **THEN** ゴーストスロットは表示されない（同一グループに挟まれた区間はランのハッチとして可視化される）

### Requirement: ゴーストからのワンクリック記録
ゴーストスロットのクリックで、開始/終了が該当区間でプリフィルされた記録ポップオーバー（時刻入力・カテゴリチップ・自由メモ）を開く SHALL。確定すると MANUAL エントリとして登録され、再描画後はゴーストが消えて記録済みブロックに置き換わる SHALL。

#### Scenario: ゴーストを昼食として記録する
- **WHEN** 「＋ 未記録 12:42–13:05」のゴーストをクリックし、カテゴリ「昼食」を選んで追加する
- **THEN** 12:42–13:05 の MANUAL エントリが登録され、タイムライン上でゴーストが「昼食」ブロックに置き換わる

### Requirement: ドラッグによる任意区間記録の存置
空きレーン領域の上下ドラッグによる任意区間の記録機能は維持する SHALL（スナップ規則・確定ポップオーバーは従前どおり）。ラン（ハッチ含む全スパン）およびゴーストスロットの上ではドラッグ記録を開始しない SHALL。

#### Scenario: ゴースト外の空き領域をドラッグで記録する
- **WHEN** ゴーストのない空き領域を上下にドラッグして離す
- **THEN** 従前どおり記録ポップオーバーが開き、任意区間を記録できる

#### Scenario: ラン上のドラッグは無効
- **WHEN** ランのハッチスライス上でマウスダウンする
- **THEN** ドラッグ記録は開始されない

### Requirement: 閾値 `away_min_seconds` の設定項目化
システムは設定値 `away_min_seconds`（既定 600 秒）を持ち、設定画面から分単位で変更できる SHALL。サーバーのギャップ抽出、クライアントのラン結合、拡張の復帰通知は、すべてこの単一の値を参照する SHALL。

#### Scenario: 閾値を下げるとゴーストが増える
- **WHEN** 設定画面で閾値を10分から5分に変更してタイムラインを再表示する
- **THEN** 5分以上10分未満の未カバー区間もゴーストスロットとして表示され、同区間はラン結合の対象からも外れる

### Requirement: ディープリンクからの記録ドラフト起動
ダッシュボードは URL ハッシュ `#timeline?from=<epoch ms>&to=<epoch ms>` を受け付け、該当日のタイムラインを開いたうえで、当該区間がプリフィルされた記録ポップオーバーを自動表示する SHALL。パラメータは消費後に URL から除去し、リロードで再発火させない SHALL。

#### Scenario: 通知経由で記録ポップオーバーが開く
- **WHEN** `#timeline?from=...&to=...`（12:42–13:05 に相当）付きの URL でダッシュボードを開く
- **THEN** タイムラインタブが表示され、12:42–13:05 がプリフィルされた記録ポップオーバーが自動で開き、URL からパラメータが除去される

### Requirement: 記録カテゴリの直近使用順表示と新規登録
記録ポップオーバー（ゴースト・ドラッグ・ディープリンクの全導線で共通）のカテゴリチップは、`GET /api/categories` が返すレジストリ由来のカテゴリを提示する SHALL。提示順は、追跡中カテゴリ（採用中の `TIMELINE` 条件が一致対象とするカテゴリ）を上限に関わらず常に先頭へ固定表示 SHALL し、続けて残りのカテゴリを直近使用順に表示する SHALL。既定でチップは一定の上限件数まで表示 SHALL し、上限を超えた分は「もっと見る」操作で展開して表示できる SHALL（追跡中カテゴリは上限の対象外として常に見える）。ポップオーバーは新規カテゴリ名を入力できる入力欄を持ち、記録確定時に選択または入力されたカテゴリをサーバーへ登録（upsert）する SHALL。カテゴリ取得に失敗した場合は既定カテゴリにフォールバックして記録機能を維持する SHALL。ポップオーバーは複数カテゴリの選択に対応 SHALL し、複数選択時は同一区間の同時記録として確定する（同時記録の詳細は timeline-coactive-record を参照）。

#### Scenario: 追跡中カテゴリがチップ先頭に固定される
- **WHEN** 「掃除」を追跡中で、直近使用の上位に「掃除」が含まれない状態で記録ポップオーバーを開く
- **THEN** 「掃除」チップは上限に関わらず先頭に固定表示され、埋もれない

#### Scenario: 直近使ったカテゴリがチップ先頭付近に出る
- **WHEN** 直近に「通院」で記録した後、別のゴーストをクリックして記録ポップオーバーを開く
- **THEN** 追跡中カテゴリの後ろに、直近使用順で「通院」が前方に表示される

#### Scenario: 上限超過分は「もっと見る」で展開できる
- **WHEN** レジストリのカテゴリ数が上限を超える状態で記録ポップオーバーを開く
- **THEN** 上限までのチップと「もっと見る」が表示され、「もっと見る」で残りのチップが展開される

#### Scenario: 新規カテゴリを入力して記録すると翌日も候補に出る
- **WHEN** 記録ポップオーバーで「買い物」と入力して追加する
- **THEN** 「買い物」で MANUAL エントリが登録され、以後（翌日を含む）の記録ポップオーバーのチップに「買い物」が現れる

#### Scenario: カテゴリ取得失敗時も記録できる
- **WHEN** カテゴリ取得 API が失敗した状態で記録ポップオーバーを開く
- **THEN** 既定カテゴリのチップが表示され、記録は従来どおり実行できる

### Requirement: 記録ポップオーバーの「記録」操作への常時到達性
記録ポップオーバー（ゴースト／ドラッグ／ディープリンクの全導線で共通）は、内容が縦に伸びても「記録」ボタン等の主要操作へ常に到達できる SHALL。パネルはヘッダー（「空き時間を記録」タイトル領域）をドラッグして画面内を移動できる SHALL。パネルはビューポート高に収まる `max-height` を持ち、内容がそれを超える場合は内部スクロールで下端の操作を露出する SHALL。初期表示位置は、可能な限り「記録」ボタンを含む全高がビューポート内に収まる位置へクランプされる SHALL。

#### Scenario: 内容が縦に伸びても記録ボタンに到達できる
- **WHEN** カテゴリチップを「もっと見る」で展開し、ポップオーバーの内容がビューポート高を超える
- **THEN** パネルは内部スクロールで下端まで辿れ、「記録」ボタンが画面外に見切れることなく操作できる（ブラウザの縮小操作を要しない）

#### Scenario: ヘッダーを掴んでポップオーバーを移動できる
- **WHEN** ポップオーバーのヘッダー（「空き時間を記録」タイトル領域）を押してドラッグする
- **THEN** パネルがポインタに追従して移動し、記録操作が画面下端に隠れている場合でも見える位置へ寄せられる

#### Scenario: ドラッグ移動中もポップオーバーは閉じない
- **WHEN** ヘッダーからドラッグを開始してポインタをパネル外まで動かして離す
- **THEN** バックドロップのクリック扱いにならずポップオーバーは開いたままで、入力内容は保持される

### Requirement: 記録ポップオーバーの Enter による確定
記録ポップオーバーは Enter キーで記録を確定できる SHALL。時刻入力欄（開始／終了）での Enter は記録を実行する SHALL。カテゴリ入力欄での Enter は、入力文字がある場合はカテゴリ追加（従前どおり選択集合へ加える）を行い、入力が空の場合は記録を実行する SHALL。IME 変換確定の Enter（`e.isComposing` が真、または keyCode 229）は記録・追加のいずれも行わず無視する SHALL。記録処理中に「記録」ボタンが `disabled` の間の Enter は無視し、二重送信を発生させない SHALL。記録実行時は、未追加のまま入力欄に残っているカテゴリ名も取り込む（従前の確定挙動）SHALL。

#### Scenario: 時刻欄で Enter を押すと記録される
- **WHEN** カテゴリを1つ以上選択した状態で、開始または終了の時刻入力欄にフォーカスして Enter を押す
- **THEN** その区間・カテゴリで記録が確定し、ポップオーバーが閉じてタイムラインが再描画される

#### Scenario: カテゴリ欄は文字の有無で挙動が分かれる
- **WHEN** カテゴリ入力欄に「買い物」と入力して Enter を押す
- **THEN** 「買い物」が選択集合へ追加され入力欄がクリアされる（記録は実行されない）
- **WHEN** 続けてカテゴリ入力欄が空の状態で Enter を押す
- **THEN** 選択済みカテゴリで記録が確定する

#### Scenario: IME 変換確定の Enter では記録も追加もされない
- **WHEN** カテゴリ入力欄で日本語を変換中に変換確定の Enter（`isComposing`）を押す
- **THEN** カテゴリ追加も記録確定も発生せず、変換だけが確定する

#### Scenario: 記録処理中の Enter は二重送信しない
- **WHEN** Enter で記録を開始し、非同期処理で「記録」ボタンが `disabled` の間にもう一度 Enter を押す
- **THEN** 2回目の記録は実行されず、エントリは重複して作成されない
