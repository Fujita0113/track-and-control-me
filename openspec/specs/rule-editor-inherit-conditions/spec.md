# rule-editor-inherit-conditions Specification

## Purpose

ルール編集画面で未来日ルールを新規作成する際、対象日が継承している実効ルールの条件をエディタへプリロードし、目標採用条件の欠落による `GoalLockError` を防ぐ。

## Requirements

### Requirement: 未来日ルールの新規作成は継承中の実効条件をプリロードする

ルール編集画面の「＋ ルールを作成」で対象日（既存ルールがある場合は翌日、ルール皆無なら今日）を開くとき、その対象日に**明示ルールが無い**場合、システムはエディタの初期条件を空にせず、対象日が現在継承している実効ルール（`effective_date` が対象日より前で最も新しいルールセット）の全条件でプリロード SHALL する。プリロードした条件は `condition_key` を保持し、目標が採用中のもの（ジャンル固定）は既存のロック表示（🔒・種別/グループ/カテゴリ変更不可・行削除不可）を適用 SHALL する。対象日に明示ルールが既に存在する場合はその明示ルールの条件を用い（従来どおり）、実効ルールが皆無の真の初期状態では空＋既定条件で開く（従来どおり）。

#### Scenario: 継承条件を保持したまま新規条件を追加できる

- **WHEN** 今日の実効ルールに `total_work` / `planning:tomorrow_planned` / `timeline:掃除` があり、翌日に明示ルールが無い状態で「＋ ルールを作成」（対象=翌日）を開き、開発グループの閾値条件を1つ追加して保存する
- **THEN** エディタは継承条件（`total_work` / `planning:tomorrow_planned` / `timeline:掃除`）をプリロードした状態で開き、保存される翌日ルールには継承条件＋追加した開発グループ条件がすべて含まれ、目標のジャンル固定条件が欠落しないため `GoalLockError` は発生しない

#### Scenario: 継承した目標採用条件はロック表示される

- **WHEN** 継承条件のうち `planning:tomorrow_planned` と `timeline:掃除` が進行中/開始前の目標に採用されている状態で新規作成エディタを開く
- **THEN** これらの条件行は🔒ジャンル固定として表示され、種別・グループ・カテゴリの変更と行削除が不可になる（閾値は理由入力を伴って変更可能）

#### Scenario: 対象日に明示ルールがある場合は明示ルールを用いる

- **WHEN** 対象日（翌日）に既存の明示ルールセットが存在する状態で「＋ ルールを作成」を開く
- **THEN** エディタはその明示ルールの条件で開き、継承ルールでの上書きは行わない

#### Scenario: 実効ルールが皆無なら従来どおり空で開く

- **WHEN** ルールセットが1件も存在しない真の初期状態で「＋ ルールを作成」（対象=今日）を開く
- **THEN** エディタは既定条件（`TOTAL_WORK`）1件だけで開き、継承プリロードは行わない
