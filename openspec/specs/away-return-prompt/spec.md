# away-return-prompt Specification

## Purpose

拡張（MV3 Service Worker）が離席からの復帰を検知し、閾値以上の離席について「離席を記録しませんか？」の OS 通知を表示する。通知クリックでタイムラインの記録ポップオーバーへ直行できる。通知は補助経路であり、確実な回収経路はタイムラインのゴーストスロットが担う。

## Requirements

### Requirement: 復帰検知と通知
拡張は idle/locked から active への復帰を検知し、離席時間（復帰時刻 − 最終 active 時刻）が閾値 `away_min_seconds` 以上の場合に OS 通知（`chrome.notifications`）を表示する SHALL。通知は離席の時間帯（HH:MM–HH:MM）と分数を含む SHALL。閾値未満の離席では通知しない SHALL。

#### Scenario: 閾値以上の離席から復帰すると通知される
- **WHEN** 23分間（閾値10分以上）離席して PC 操作を再開する
- **THEN** 「離席を記録しませんか？」の OS 通知が離席の時間帯・分数付きで表示される

#### Scenario: 短い離席では通知されない
- **WHEN** 6分間（閾値未満）離席して操作を再開する
- **THEN** 通知は表示されない

### Requirement: スリープ・再起動からの復帰の補足
拡張は Service Worker の初期化時（ブラウザ起動・スリープ復帰後の wake を含む）にも最終 active 時刻からの経過を判定し、閾値以上であれば同様に通知する SHALL。同一の離席区間に対する通知は1回に抑止する SHALL。

#### Scenario: PC スリープからの復帰で1回だけ通知される
- **WHEN** PC を40分スリープさせて復帰し、その後拡張のイベントが複数回発火する
- **THEN** 当該離席区間への通知は1回だけ表示される

### Requirement: 通知からの記録導線
通知のクリックで、ダッシュボードのタイムラインを離席区間付きのディープリンク（`#timeline?from=<ms>&to=<ms>`、接続先は設定済み `wsPort`）で新しいタブに開く SHALL。

#### Scenario: 通知クリックで記録ポップオーバーに直行する
- **WHEN** 離席通知をクリックする
- **THEN** `http://127.0.0.1:<wsPort>/#timeline?from=...&to=...` が開き、該当区間がプリフィルされた記録ポップオーバーが表示される

### Requirement: 閾値のサーバー同期
閾値はサーバー設定 `away_min_seconds` を権威とし、WS ハンドシェイクの `welcome` メッセージで拡張へ配布する SHALL。拡張は受領値を永続化して使用し、未受領（未接続・旧サーバー）の場合は契約既定値（600 秒）にフォールバックする SHALL。

#### Scenario: 設定変更が再接続後の拡張に反映される
- **WHEN** 設定画面で閾値を15分へ変更し、拡張が WS を再接続する
- **THEN** welcome で新しい閾値を受領し、以後15分未満の離席では通知されない

### Requirement: サーバー停止時のフォールバック
通知機能はサーバー接続に依存せず動作する SHALL（閾値は最後に受領した値または既定値）。通知の失敗・無視・サーバー停止中のクリック不能があっても、離席区間は後からタイムラインの未記録ゴーストスロットとして回収可能である SHALL。

#### Scenario: サーバー停止中の離席も後から記録できる
- **WHEN** サーバー停止中に30分離席して復帰する（サンプルは拡張側にバッファされる）
- **THEN** 通知は表示され、サーバー再開後にタイムラインを開くと該当区間が「＋ 未記録」ゴーストとして表示され記録できる
