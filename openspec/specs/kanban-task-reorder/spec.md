# kanban-task-reorder Specification

## Purpose

カンバンの並べ替え可能な列（保留・未着手・進行中）で、カードを同一列内ドラッグ&ドロップして任意位置へ並べ替え、その順序を各タスクの `sort_order` として堅牢・一意に永続化する。ドラッグ中は挿入位置インジケータを表示し、新規カードには列末尾ランクを付与、既存の未整列列（全 0 など）は並べ替え時に連番へ正規化する。列間移動もドロップ位置に応じたインデックスへ配置し、完了列（DONE）は完了演出＋アーカイブ専用として並べ替え対象外とする。

## Requirements

### Requirement: 列内ドラッグ&ドロップによる並べ替え

システムは、カンバンの並べ替え可能な列（保留・未着手・進行中）において、カードを**同一列内でドラッグ&ドロップして任意の位置へ並べ替え**できるようにする SHALL。ドロップ位置に応じた挿入先インデックスを計算し、当該列内のカード順序をその順に更新する SHALL。同一列内ドロップは（従来の no-op ではなく）並べ替えとして扱う SHALL。

#### Scenario: 未着手列でカードを上へ移動

- **WHEN** ユーザーが「未着手」列の下方にあるカードを、同列の上位のカードの上へドラッグ&ドロップする
- **THEN** そのカードは指定位置に挿入され、列内の並び順が更新される
- **AND** 更新後の並び順が永続化され、再読み込み後も維持される

#### Scenario: 同じ位置へドロップしても破綻しない

- **WHEN** ユーザーがカードをつかみ、ほぼ同じ位置へドロップする
- **THEN** 並び順は実質的に変化せず、エラーや重複表示は発生しない

### Requirement: 挿入位置インジケータ

システムは、ドラッグ中に**ドロップされる挿入位置を示すインジケータ（プレースホルダ）**を表示する SHALL。インジケータはポインタ位置に応じて各カードの中点を基準に更新され、列全体ハイライトだけに依存しない SHALL。HTML5 ドラッグ中の再描画がドラッグを破壊しないよう、インジケータは全体再描画（renderAll）ではなく直接 DOM 操作で更新する SHALL。

#### Scenario: ドラッグ中に挿入位置が示される

- **WHEN** ユーザーがカードをドラッグし、列内のカード間へポインタを移動する
- **THEN** そのカード間に挿入位置インジケータが表示される
- **AND** ポインタを別のカード間へ動かすとインジケータの位置が追従する

#### Scenario: ドラッグ終了でインジケータが消える

- **WHEN** ユーザーがドロップまたはドラッグを中止する
- **THEN** 挿入位置インジケータは除去される

### Requirement: 並び順の永続化と表現

システムは、列内の並び順を各タスクの `sort_order` として永続化する SHALL。一覧取得時は `status, sort_order, id` の順で並べ、フロントは取得順を列内順序として描画する SHALL。並べ替えの保存はサーバへ反映され、複数カードの `sort_order` が変わる場合も整合した状態で永続化される SHALL。

#### Scenario: 並び順がサーバへ保存される

- **WHEN** ユーザーが列内でカードを並べ替える
- **THEN** 影響を受けるカードの `sort_order` が更新され、サーバに保存される
- **AND** 別のクライアント/再読み込みでも同じ並び順で表示される

### Requirement: 新規カードへの末尾ランク付与

システムは、新規カード作成時に、追加先の列（既定では未着手など作成時の状態）の**末尾に来る `sort_order`** を明示的に付与する SHALL。全カードが `sort_order = 0` で衝突する状態を作らない SHALL。

#### Scenario: 新規カードは列の末尾に追加される

- **WHEN** ユーザーが列にカードを新規追加する
- **THEN** そのカードは当該列の既存カードより後ろ（末尾）に配置される
- **AND** そのカードには他カードと衝突しない `sort_order` が付与される

### Requirement: 列単位の順序正規化

システムは、`sort_order` が未設定・衝突（全 0 など）している既存カードを含む列に対し、並べ替え操作時に列単位で `sort_order` を連番（0,1,2,…）へ正規化できる SHALL。正規化は列内の視覚的な並び順を保持したまま行う SHALL。

#### Scenario: 既存の未整列カードを並べ替えると正規化される

- **WHEN** すべての `sort_order` が 0 の既存カードが並ぶ列で、ユーザーが並べ替えを行う
- **THEN** 当該列のカードは現在の表示順を保ったまま連番の `sort_order` に振り直される
- **AND** その後の並べ替えでも一意な順序が維持される

### Requirement: 列間移動時の挿入位置指定

システムは、カードを別の列へドロップする際、状態（列）を更新するだけでなく、**ドロップ位置に応じたインデックス**へ挿入する SHALL。移動先列内の該当位置に配置され、末尾固定にはしない SHALL。完了列（DONE）へのドロップは従来どおり完了演出＋アーカイブ扱いとし、本挿入位置指定の対象外とする SHALL。

#### Scenario: 別列の中間位置へ移動

- **WHEN** ユーザーがカードを別の列の中間位置へドロップする
- **THEN** カードの状態が移動先の列に更新される
- **AND** カードは移動先列内のドロップ位置に挿入され、その並び順が永続化される

#### Scenario: 完了列は並べ替え対象外

- **WHEN** ユーザーがカードを完了列へドロップする
- **THEN** 従来どおり完了として記録され、完了演出の後にアクティビティログへ追加される
- **AND** 完了列内での並べ替えインジケータや位置指定は行われない
