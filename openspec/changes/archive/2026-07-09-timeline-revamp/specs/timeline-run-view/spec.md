# timeline-run-view

タイムラインの AUTO ブロック描画をラン（表示レイヤーでの断片結合）ベースに刷新し、ガター・ラベル・カラム割当のノイズを除去する。データ層（session / creditedMs / gaps 計算）は変更しない。

## ADDED Requirements

### Requirement: 同一グループ断片のラン結合表示
タイムラインは、同一 `stableGroupId` の隣接する AUTO ブロック a, b を、次の両条件を満たす場合に限り1つのラン（単一ブロック）として描画する SHALL:
1. `b.startAt - a.endAt` が閾値 `away_min_seconds` 未満
2. 区間 `(a.endAt, b.startAt)` に他の描画ブロック（他グループの AUTO、または MANUAL エントリ）が重ならない

結合は描画時のみ行い、サーバーの session データ・`creditedMs`・`gaps` 計算には影響しない SHALL。

#### Scenario: 微小離席を挟む同一グループ断片は1ランになる
- **WHEN** 「面接」の AUTO ブロックが 10:47–10:55 と 11:01–11:22 に存在し、間隔6分が閾値（10分）未満で、間に他ブロックがない
- **THEN** 10:47–11:22 の単一ブロックとして描画され、タイトル「面接」と時間帯ラベルは1回だけ表示される

#### Scenario: 他グループの作業を挟む場合は結合しない
- **WHEN** 「面接」ブロックの間（5分）に「ブログ投稿」の AUTO ブロックが存在する
- **THEN** 「面接」は2つの独立したブロックとして描画され、「ブログ投稿」の時間がハッチとして描かれることはない

#### Scenario: 記録済み離席（MANUAL）を挟む場合は結合しない
- **WHEN** 「面接」ブロックの間に MANUAL エントリ「昼食」が存在する
- **THEN** 「面接」は2つの独立したブロックとして描画される

#### Scenario: 閾値以上の間隔では結合しない
- **WHEN** 同一グループのブロック間隔が23分（閾値10分以上）
- **THEN** 2つの独立したブロックとして描画される

### Requirement: ラン内離席のハッチ表示
ランは内部の間隙（結合で吸収した離席区間）をハッチ（斜線）スライスとして実時間位置に描画する SHALL。スライスの描画高さは最低 4px を保証し、高さ 16px 以上の場合はスライス内に「離席 n分」ラベルを表示、未満の場合はツールチップで時間帯と分数を提示する SHALL。

#### Scenario: 吸収された離席がハッチとして見える
- **WHEN** 10:47–11:22 のランが 10:55–11:01 の間隙を吸収している
- **THEN** ラン内の該当位置（10:55–11:01）にハッチスライスが描画される

#### Scenario: 極小の離席も視認できる
- **WHEN** 吸収された間隙が2分（描画高さ 2.4px 相当）
- **THEN** スライスは最低高さ 4px で描画され、ツールチップで「HH:MM–HH:MM（2分）」を確認できる

### Requirement: ブロックラベルの一回表示と「・同時n」サフィックス廃止
ランおよび単独ブロックのタイトル・時間帯ラベルはブロックにつき1回だけ表示する SHALL。同時オープン数を示す「・同時n」テキストサフィックスは表示しない SHALL。同時性は列分割レイアウト（横並び）自体が伝達する。

#### Scenario: 同時オープン2グループの表示
- **WHEN** 「ブログ投稿」と「振り返り」が同一時間帯に同時オープンだった
- **THEN** 2つのブロックが横並びのカラムで描画され、いずれのタイトルにも「・同時2」は付かない

### Requirement: 詳細ポップオーバーのクレジット内訳表示
AUTO ラン/ブロックのクリックで開く詳細ポップオーバーは、時間帯（スパン）、実働クレジット（結合した断片の `creditedMs` 合計）、同時に開いていたグループ名の一覧（均等割の注記付き）、離席内訳（回数・合計分数・各区間）を表示する SHALL。

#### Scenario: divide-by-N のクレジットが確認できる
- **WHEN** 60分間2グループ同時オープンだったブロックをクリックする
- **THEN** ポップオーバーに実働クレジット約30分と、同時に開いていたグループ名、均等割である旨の注記が表示される

#### Scenario: ランの離席内訳が確認できる
- **WHEN** 2つの間隙を吸収したランをクリックする
- **THEN** ポップオーバーに「離席 2回・合計 n分」と各区間の時間帯が表示される

### Requirement: ガターの正時ラベル限定
時刻ガターは正時（HH:00）ラベルのみを表示する SHALL。ブロック断片境界の時刻ラベルおよびレーンの境界破線は描画しない SHALL。

#### Scenario: 断片が多い日でもガターが整然としている
- **WHEN** 1日に10個以上のブロック断片が存在する日を表示する
- **THEN** ガターには正時ラベルのみが等間隔に表示され、ラベル同士の重なりが発生しない

### Requirement: カラム割当の安定化
列分割レイアウトは決定的である SHALL。クラスタ内のカラム割当では、同一グループが直前のクラスタで使用したカラムが空いていればそのカラムへ再割当し、衝突時のみ first-fit へフォールバックする SHALL。

#### Scenario: 同一グループが連続クラスタで同じ側に留まる
- **WHEN** 「面接」が隣接する2つのクラスタに出現し、直前クラスタで使用したカラムが次クラスタでも空いている
- **THEN** 「面接」は両クラスタで同じカラム位置に描画される
