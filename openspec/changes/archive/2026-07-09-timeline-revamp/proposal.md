## Why

タイムライン画面（ui-revamp の `timeline-calendar`）は、短時間離席のたびに同一グループのブロックが分断されてタイトルが連呼される（「面接」が上にも下にも）、左ガターに断片境界の時刻ラベルが密集して重なる、という2つの構造要因で「何時から何時に何をしていたか」が一目で読めない。また離席の記録手段がドラッグ操作のみで、作業中ダッシュボードを開いていない運用では離席が未記録のまま放置される。

## What Changes

- **ラン描画（表示レイヤーの断片結合）**: 同一グループの AUTO ブロック断片を、間のギャップが閾値未満 **かつ そのギャップを他グループのブロックが占有していない** 場合に限り、描画時に1つの「ラン」へ結合する。タイトル・時間帯ラベルはランに1回だけ表示し、ラン内部の微小離席はハッチ（斜線）スライスとして描く。データ（session / creditedMs / gaps 計算）は無変更 — `session_coalesce_seconds` は上げない（computeGaps が coalesce 後ブロックを参照するため、上げるとギャップデータ自体が消える副作用がある）。
- **切り替えとの区別**: 面接→ブログ5分→面接のようなグループ切り替えはラン結合しない。結合すると他グループ作業時間が離席ハッチとして描かれ嘘になるため。
- **ガター簡素化**: 時刻ガターは正時ラベルのみとし、断片境界の目盛りラベル（`boundaryMinutes` のガター出力）を廃止する。正確な開始/終了はラン自身のラベルと詳細ポップオーバーが担う。
- **「・同時n」サフィックス廃止**: 横並びの列分割レイアウト自体が同時性を伝えるため、ブロック名のテキストサフィックスは冗長として削除。divide-by-N クレジットの内訳（均等割で実働○分）はクリック時の詳細ポップオーバーへ移す。並列カラム表示と divide-by-N の意味論（同時に開いているグループは並行作業・主従なし）は維持する。
- **カラム安定化**: ラン化により断片が長いランへまとまり、クラスタ境界（幅・カラム位置が変わる地点）が激減して同一グループが左右へフリップする現象が減る。加えてクラスタ内のカラム割当を開始時刻順で安定させる。
- **未記録ギャップのゴーストスロット**: サーバーが計算済みで現状 UI 未使用の `gaps` を「＋ 未記録 HH:MM–HH:MM (n分)」のゴーストとしてレーンに描画し、クリックで既存のカテゴリチップ式ポップオーバー（区間プリフィル済み）を開く。ドラッグ記録は任意区間用として残すが、主経路はゴーストのワンクリックにする。
- **復帰時 OS 通知（離席記録プロンプト）**: 拡張の Service Worker が離席（idle/locked/サンプル途絶）からの復帰を検知し、離席時間が閾値以上なら chrome.notifications で「離席を記録しませんか？」を通知する。クリックでダッシュボードのタイムラインが開き、該当区間の記録ポップオーバーがプリフィル済みで表示される（URL パラメータによるディープリンク）。サーバー停止中はゴーストスロットが確実なフォールバックとなる。
- **閾値の一元化**: 「サーバーのギャップ抽出（現状 300s ハードコード）」「クライアントのラン結合」「通知を出す最小離席時間」を単一の設定値（既定 10 分）に統一し、設定画面から変更可能にする。閾値未満＝作業の呼吸として吸収、以上＝記録すべき離席、という一貫ルールにする。

## Impact

- **フロントエンド（主）**: `server/static/js/timeline.js` — 描画部の全面書き換え（ラン結合・ゴーストスロット・ガター・カラム安定化・URL パラメータからのドラフト起動）。`server/static/css/app.css` — ラン/ハッチ/ゴースト様式の追加。`server/static/js/settings.js` — 閾値設定項目の追加。
- **サーバー（小）**: `server/src/services/timeline.ts` — `computeGaps` の 300s 固定を設定値へ。`server/src/db/migrations.ts` — 閾値の config カラム追加（既定 600s）。`server/src/api/index.ts` — 設定 API の許可キーへ追加。
- **拡張（中）**: `extension/src/sw.ts` — 離席開始/復帰の追跡と通知発火・通知クリックでダッシュボードを開くハンドラ。`extension/manifest.json` — `notifications` 権限の追加。
- **無変更**: 集計（divide-by-N・gapCap・day boundary）、session/activity_log_entry スキーマ、`/api/timeline/*` のレスポンス形状（gaps の件数は閾値変更の影響のみ）。
- **リスク**: (1) ラン結合とゴーストの閾値がズレると「どこにも現れない帯」が生じる → 一元化で構造的に排除。(2) 通知の過剰発火（短い離席の連続） → 閾値未満は通知しない・ゴーストにも出ない設計で抑制。(3) サーバー停止中の通知クリックはページが開けない → 通知は即時記録のショートカット、回収はゴーストが担う二段構え。
