## Why

いまは目標もルール変更も「翌日から」に固定されている。これは「当日に解錠条件を骨抜きにするゲーミング」を抑えるための凍結ポリシーが目的で、**緩める方向**を当日に効かせない、というのが本質だった。ところが「今日から始めたい」「今日のうちにもう一本ルールを足したい」という **自分を厳しくする方向** まで一律に翌日送りになっていて、思い立った日に踏み込めない。緩める方向だけ翌日のまま据え置き、厳しくする方向は当日に許すことで、凍結ポリシーの目的を保ったまま身動きを取れるようにする。

あわせて、目標の記録（振り返りタブの目標ごとの日記コーナー）が実際どう見えるのかを、デモモードから確認しづらい（デモ入場直後は目標が「開始前」で期間外扱いになり空表示になる）。デモでその見え方を確認できるようにする。

## What Changes

- **目標の開始日を「今日から／明日から」の選択式にする**（既定は「今日から」）。従来の「翌日固定」をやめる。期間は開始日から30日固定（`end_day = start_day + 29`）のまま。
  - 「今日から」の目標は当日を Day 1 として即座に「進行中」になり、採用実践は **開始日（＝当日）の実効ルール** に対して検証・スナップショットする。
  - ジャンル固定（採用中条件を残期間から欠けさせる編集の拒否）を `start_day` 起点に広げる（当日開始なら当日から保護）。
- **当日中の新規ルール条件の追加を許可する**（`same-day-rule-additions`）。
  - 当日に条件を **新しく追加** できる。その日に追加した条件は、同日中は自由に編集（閾値変更等）・削除できる。
  - 既存（前日までに凍結済み）の条件は当日いじれない（削除・引き下げは従来どおり翌日から）。＝当日のゲートが day 開始時点の基準より **緩くなることはない**（厳しくする方向のみ当日反映）。
  - 当日追加した条件を目標が採用した場合は、その条件は同日でも削除できなくなる（ジャンル固定に準じる）。
- **インライン条件作成（`goal-inline-condition`）を当日開始の目標にも対応させる**。当日開始なら新規条件は **当日ルール** へ追記して採用する（翌日開始なら従来どおり翌日ルールへ）。
- **デモモードの振り返りで各目標の記録の見え方を確認できるようにする**。デモ入場直後（開始前）でも、目標ごとの記録コーナーがどう見えるかを閲覧できる状態にする。

## Capabilities

### New Capabilities
- `same-day-rule-additions`: 当日ルールへの新規条件の追加を許可する凍結ポリシーの例外。当日追加分は同日中に限り自由に編集・削除でき、既存の凍結条件には触れられない（厳しくする方向のみ当日反映）。目標が採用した当日追加条件は同日でも削除不可になる。

### Modified Capabilities
- `goal-challenge`: 開始日を「今日／明日」の選択式（既定=今日）に変更。翌日固定をやめ、採用実践の検証・スナップショットとジャンル固定の起点を `start_day`（当日開始なら当日）に据える。削除は作成当日のみ・状態導出・30日固定は不変。
- `goal-inline-condition`: インライン新規条件の追記先を、目標の開始日に応じて当日ルール（当日開始）／翌日ルール（翌日開始）へ振り分ける。
- `demo-mode`: デモの振り返りで、目標ごとの記録コーナーの見え方を（開始前を含め）確認できるようにする。

## Impact

- **サーバー**
  - `server/src/rules/rules.ts`: 凍結ポリシーの例外拡張（`canWriteTodayRule` 相当の判定に「既存凍結条件は不変・新規追加は当日可」を追加）、`upsertFutureRuleSet` / `deleteRuleSet` の当日編集判定、`assertGoalsSatisfied` のロック起点（`max(today, start_day)` 系）と当日追加条件の採用ロック。
  - `server/src/services/goals.ts`: `start_day` を選択入力（今日／明日）に、`GOAL_DAYS` 期間算術、採用候補・検証を `start_day` 実効ルールへ、インライン条件の追記先分岐。
  - `server/src/api/`: 目標作成 API に開始日の選択パラメータ、ルール PUT/DELETE の当日追加パス。
- **拡張フロント**
  - `server/static/js/goals.js`: 作成 UI に開始日の選択（今日／明日）、採用候補の解決元。
  - `server/static/js/rules.js`: 当日の条件追加動線（既存条件は当日ロック表示のまま）。
  - `server/static/js/reflection.js`: デモ時の目標記録コーナーの表示。
  - `server/src/services/demo-seed.ts` / `demo.js`: デモの振り返りで目標記録の見え方を確認できるサンプル状態。
- **契約/スキーマ**: 既存 `daily_rule_set` / `rule_condition` / `goal` カラムで表現可能か design で確定（当日追加条件の識別＝`created_at` 由来の day 一致で判定できる見込み。新スキーマ追加は最小限）。
