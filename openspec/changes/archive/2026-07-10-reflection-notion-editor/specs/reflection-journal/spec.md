## ADDED Requirements

### Requirement: インライン・ライブ Markdown エディタ

振り返りタブの本文入力は、単一の `contenteditable` 要素による**インライン・ライブ Markdown エディタ**でなければならない（MUST）。Markdown 構文マーカーは淡色（`#b4bdcc`）で表示しつつ、同じ行に整形結果をインライン描画する（Obsidian/Notion ライブプレビュー方式）。従来の textarea＋別ペインプレビューは提供してはならない（MUST NOT）。

本文は Markdown テキストとして保存し、`reflection_entry.content` の意味（`content.trim().length > 0` を「振り返り完了」とみなす PLANNING シグナル）を変えてはならない（MUST NOT）。

#### Scenario: 見出しのインライン整形
- **WHEN** ユーザーが行頭で `# 見出し` と入力する
- **THEN** `#` マーカーは淡色で残り、続くテキストは見出しサイズ（`#`=29px/`##`=24px/`###`=20px/`####`=17px/`#####`=16px/`######`=15px）で同じ行に整形描画される

#### Scenario: 強調・コードのインライン整形
- **WHEN** ユーザーが `**太字**`・`*斜体*`・`` `コード` ``・`~~打消~~` を入力する
- **THEN** それぞれのマーカーは淡色で残り、囲まれたテキストは太字（`#111a26`/weight 800）・斜体・インラインコード（等幅・背景 `#eef1f7`）・打消線として同じ行に描画される

#### Scenario: IME（日本語）入力を壊さない
- **WHEN** ユーザーが IME で日本語を変換中（composition 中）に入力する
- **THEN** 変換確定まで再描画を抑止し、確定後に再描画してもキャレット位置が保持される

### Requirement: Notion 風キーボード挙動

エディタは以下の Notion 風キーボード挙動を提供しなければならない（MUST）。いずれも「現在行・桁の算出 → raw テキスト変換 → 再描画 → キャレット再設定」で一貫して動作する。

#### Scenario: 箇条書きの Enter 継続
- **WHEN** `- `/`*`/`+`/`1.` のリスト項目に本文があり、行末で Enter を押す
- **THEN** 同じマーカー（番号リストは連番を +1、タスクは `- [ ] `）で始まる新しい行が挿入され、キャレットは新マーカー直後に置かれる

#### Scenario: 空リスト項目で抜ける
- **WHEN** マーカーのみで本文が空のリスト項目で Enter を押す
- **THEN** マーカーが除去されて空の段落になり、新しい箇条書きは追加されない

#### Scenario: コードフェンス自動クローズ
- **WHEN** ユーザーがある行を ` ``` `（言語指定付きも可）にする
- **THEN** その行の下に閉じ ` ``` ` 行が自動挿入され、キャレットは開き行と閉じ行の間の空行に置かれる

#### Scenario: todo ショートハンド
- **WHEN** ユーザーが行頭（または `- ` 直後）で `[ ]`（または `[]`）に続けてスペースを入力する
- **THEN** その行は `- [ ] ` のタスク項目（チェックボックス付き）に変換される

#### Scenario: Backspace 1 回でマーカー削除
- **WHEN** 空のリスト項目または空の todo 項目（マーカーのみ）で Backspace を 1 回押す
- **THEN** マーカー接頭辞ごと削除されて空行になる（前行との結合は起きない）

#### Scenario: チェックボックスのクリックトグル
- **WHEN** ユーザーが描画済みタスク項目のチェックボックスをクリックする
- **THEN** raw テキストの `[ ]`⇄`[x]` がトグルされ、エディタが再描画される

### Requirement: 5 段階「気分」入力

満足度は設計の **5 段階ピル**（いまひとつ／まあまあ／ふつう／良い／とても良い）で入力できなければならない（MUST）。内部的には satisfaction 1..5 として保存する。

#### Scenario: 気分を選択・解除する
- **WHEN** ユーザーが気分ピルのいずれかを選択する
- **THEN** そのピルが選択状態（背景 `#3b5bb5`・白字）になり satisfaction 1..5 に対応する
- **WHEN** 選択中と同じピルを再度クリックする
- **THEN** 選択が解除され satisfaction は未設定（null）になる

### Requirement: 保存挙動

本文と気分は手動の「保存する」ボタンで保存でき、保存後に「保存しました」を一時表示しなければならない（MUST）。加えて、日付切替・過去エントリ選択・タブ離脱時に未保存の変更を自動フラッシュしなければならない（MUST）。

#### Scenario: 手動保存
- **WHEN** ユーザーが「保存する」ボタンを押す
- **THEN** `PUT /api/reflection/:date` に本文（Markdown）と satisfaction が送信され、成功時に「保存しました」が約 2 秒表示される

#### Scenario: 離脱時フラッシュ
- **WHEN** 未保存の変更がある状態で、対象日を切り替える／過去エントリを開く／別タブへ移動する
- **THEN** 移動前に現在の内容が自動保存されてからロード/離脱する

### Requirement: 過去の振り返り一覧（抜粋付き）

右レールの「過去の振り返り」は、日付・気分ラベル・本文の 2 行抜粋を表示しなければならない（MUST）。`GET /api/reflections` は Markdown 記号を除去した先頭 ~80 字の `excerpt` を各エントリに含める。本文全文は一覧レスポンスに含めない（軽量維持）。

#### Scenario: 抜粋を表示しクリックで開く
- **WHEN** 保存済みの振り返りが存在する
- **THEN** 各エントリに日付（等幅）・気分ラベル・2 行にクランプした抜粋が表示され、クリックするとそのエントリがエディタにロードされる

#### Scenario: 空状態
- **WHEN** 保存済みの振り返りが 1 件も無い
- **THEN** 「保存済みの振り返りはまだありません」と表示される

### Requirement: 設計への視覚的忠実性

振り返りタブの見た目は `ref/reflection/振り返り.dc.html` に忠実でなければならない（MUST）。配色 hex・寸法・要素アナトミーは参照のインライン style の値をそのまま `rf-*` CSS クラスへ転記する。CSP（`style-src 'self'`）のためインライン `style=` 属性を使ってはならず（MUST NOT）、静的スタイルはクラス、動的値のみ CSSOM で適用する。**フォントのみ** CSP 制約により `system-ui` フォールバックを許容する。

#### Scenario: 配色・レイアウトの一致
- **WHEN** 振り返りタブを表示する
- **THEN** ページ背景 `#eef1f6`・本文色 `#22303f`・アクセント `#3b5bb5`、左本文＋右レール（300px）の 2 カラム、エディタカード（白・border `#e4e9f1`・radius 14）と下部クローム（`#f6f8fc`・構文ヒント・文字数）が参照どおりに描画される

#### Scenario: CSP 適合
- **WHEN** 振り返りタブの DOM を検査する
- **THEN** インライン `style=` 属性は存在せず、スタイルは `rf-*` クラスと CSSOM（`element.style.*`）のみで適用されている
