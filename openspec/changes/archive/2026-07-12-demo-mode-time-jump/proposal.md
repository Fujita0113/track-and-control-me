## Why

「30日チャレンジ」は 30日経過して初めて完走レポートが見られる仕組みで、作った本人でも「続けたら最後に何が見られるのか」を今すぐ確かめる手段がない。ゴールの完成像を先に見せられれば、「完成像が描けず不安」という悩みを解消して安心して始められる。ただし通常モードで日付を自由に進められると「先の日付へ飛ばして解禁 → パスワード入手」ができてしまい、このアプリの肝（ルールを守らないとゲーム用パスワードが出ない自分への縛り）が根本から台無しになる。だから「日付を進める」体験は**デモ（お試し）モード限定・本番とは別のデータの中だけ**に閉じ込める必要がある。

## What Changes

- **お試し（デモ）モードの追加**: 通常モードとはっきり区別された、あらかじめ用意したサンプル目標・記録の中を「読むだけツアー」する体験を追加する（issue #20 で確定した A案）。
- **仮想「今日」の日付ジャンプ**: デモ中だけ上部バーに `🧪 デモモード` 帯＋日付コントロール（`＋1日 / ＋7日 / ＋30日 / 完走へ / リセット`）を出し、仮想の「今日」を前後に動かすとサンプル目標の状態（開始前 → 進行中 Day N/30 → 完走）と完走レポートの中身が連動して変わる。デモ入場直後は**開始前**から始め、順に進めて物語を頭から体験できる。
- **デモは閲覧専用（作成・削除・本物の解禁をしない）**: デモ中は目標の新規作成・削除を無効/非表示にする（履歴のない目標を作って空レポートになる不整合を根本回避）。今日タブのパスワード表示は**ダミー**（本物のパスワード生成コマンドは実行しない）。振り返り・日記は記入済みサンプルの閲覧が中心。
- **本番への非干渉を保証する分離**: デモの日付操作・サンプルデータは、本番データにも本番の解禁判定（パスワードゲート）にも一切効かない。デモ用データは本番 DB とは分離した使い捨てのサンプルとして持ち、本番の評価・reveal 経路には到達しない構造にする。
- **入り口**: 設定タブ内に「お試し（デモ）モード」トグル（`デモを開始/終了`・`サンプルをリセット`）を置く。

## Capabilities

### New Capabilities
- `demo-mode`: 通常モードから分離した「読むだけツアー」型のデモ体験。サンプル目標・記録の投入、デモ限定の仮想「今日」による日付ジャンプ（開始前→進行中→完走→レポートの連動）、デモ中の作成/削除の無効化とダミーパスワード表示、そして「日付操作とサンプルデータは本番データ・本番の解禁判定に絶対に効かせない」という分離ガードレールを定める。

## Impact

- **サーバ (`server/src`)**: デモ用の分離データセット（サンプル目標＋30日分の活動・評価・日記）を構築するモジュールと、仮想 `nowMs` を注入して読み取り専用に集計するデモ API を追加。既存の時刻依存関数は `todayKey(db, nowMs)` / `getGoalReport(db, id, nowMs)` 等が既に `nowMs` 引数を受けられるため、その土台を利用する。本番の `password/reveal`・本番 DB 書き込み経路には一切接続しない。
- **フロント (`server/static/js`)**: 上部バーのデモ帯＋日付コントロール、設定タブの入り口トグル、目標/今日/振り返り各画面のデモ対応（作成・削除の無効化、ダミーパスワード表示、サンプル連動表示）。デモ状態は中央で1つ持ち、通常モードでは全て非表示。
- **スコープ外（別 issue）**: ログイン/アカウント認証（#20 では「デモ切替」で十分）、習慣そのものを目標化（#21）、目標への画像添付 Before/After 写真（#22）、目標作成プレビュー。
