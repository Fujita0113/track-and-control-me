## ADDED Requirements

### Requirement: カテゴリ付けモードのトグル

かんばんヘッダーは、明日モードの隣に「カテゴリ付けモード」のトグルを表示しなければならない（SHALL）。この状態は明日モードと同じく localStorage のクライアント専用値として保持し、日付が変わったら OFF にリセットする（サーバの `app_config` には保存しない）。

#### Scenario: トグルをONにする
- **WHEN** ユーザーがカテゴリ付けモードのトグルをONにする
- **THEN** 状態が localStorage に保存され、以後のタスク作成でカテゴリ選択ステップが有効になる

#### Scenario: 日付が変わるとリセットされる
- **WHEN** トグルON状態で保存された日付と現在の day_key が異なる
- **THEN** モードは OFF として扱われる（明日モードと同じ日次リセット挙動）

#### Scenario: モードOFF時は従来どおり
- **WHEN** カテゴリ付けモードがOFF
- **THEN** タスク作成はカテゴリ選択ステップを挟まず、従来どおり次のタスク入力へ進む

### Requirement: 作成後のカテゴリ選択ステップ

カテゴリ付けモードがONのとき、タスクをEnterで作成した直後、本来「次のタスク入力」に移る位置に、作成したタスクのカテゴリ選択UIを表示しなければならない（SHALL）。カテゴリ選択はスキップ可能でなければならない（SHALL）。

#### Scenario: カテゴリを選んで次へ
- **WHEN** モードON時にタスクを作成し、表示されたカテゴリ選択で1つ選ぶ
- **THEN** そのカテゴリがタスクに保存され、次のタスク入力へ進む

#### Scenario: スキップして未分類のまま次へ
- **WHEN** モードON時にタスクを作成し、カテゴリ選択をEscまたは空Enterで閉じる
- **THEN** タスクは未分類（カテゴリなし）のまま保存され、次のタスク入力へ進む

#### Scenario: IME変換中のEnterで確定しない
- **WHEN** カテゴリの自由入力中に IME 変換確定のEnterを押す（`isComposing` / `keyCode 229`）
- **THEN** カテゴリ選択は確定・前進せず、変換確定のみ行われる

### Requirement: タブグループ由来のカテゴリ候補

カテゴリ選択の候補は、`GET /api/groups` が返す既知のタブグループを最近使った順（`last_seen_at` 降順）で提示し、加えて自由入力と「その他」を選べなければならない（SHALL）。

#### Scenario: 既知グループを最近順で表示
- **WHEN** カテゴリ選択を開く
- **THEN** 既知のタブグループが最近使った順で候補に並び、各候補はグループの名前と色で表示される

#### Scenario: 自由入力で新カテゴリ
- **WHEN** 候補に無い名前をユーザーが手入力して確定する
- **THEN** その名前がカテゴリ名として保存される（`category_group_id` は無し、色は無し）

#### Scenario: 「その他」を選ぶ
- **WHEN** ユーザーが「その他」を選ぶ
- **THEN** タスクは「その他」カテゴリとして保存される（グループ紐付け無し）

### Requirement: タスクへのカテゴリ保存（UUID照合＋名前色スナップショット）

タスクのカテゴリは、照合キーとしてタブグループの `stable_group_id`（UUID）を、表示用として選択時点の名前と色のスナップショットを、それぞれ保存しなければならない（SHALL）。色は列挙型やCHECK制約で縛らず、TEXT のスナップショットとして保存しなければならない（SHALL）。1タスクにつきカテゴリは最大1つとする（SHALL）。

#### Scenario: タブグループをカテゴリにする
- **WHEN** 既知タブグループ（例: 名前「競技プログラミング」・色 blue・UUID X）を選ぶ
- **THEN** タスクに `category_group_id = X`、`category_name = "競技プログラミング"`、`category_color = "blue"` が保存される

#### Scenario: グループを消してもスナップショットが残る
- **WHEN** カテゴリ付与済みタスクの元タブグループが削除される
- **THEN** タスクの `category_name` / `category_color` スナップショットは変わらず残り、カード表示に使える

#### Scenario: グループの色を後から変えても照合は切れない
- **WHEN** タスクにカテゴリを付けた後、元タブグループの色が変わる
- **THEN** タスクの照合は `category_group_id`（UUID）で維持され、`category_name` / `category_color` スナップショットは付与当時の値のまま保持される

### Requirement: カード上のカテゴリ表示

かんばんカードは、カテゴリを持つタスクに対して、カテゴリ名を色付きバッジで1つ表示しなければならない（SHALL）。

#### Scenario: 色付きバッジ表示
- **WHEN** カテゴリ（名前と色）を持つタスクのカードを表示する
- **THEN** カテゴリ名がカテゴリ色のバッジで1つ表示される

#### Scenario: 色なしカテゴリの表示
- **WHEN** 自由入力/「その他」など色を持たないカテゴリのタスクを表示する
- **THEN** バッジは色無し（中立色）で名前のみ表示される

#### Scenario: 未分類は非表示
- **WHEN** カテゴリを持たないタスクのカードを表示する
- **THEN** カテゴリバッジは表示されない

### Requirement: 集計・評価への非波及

カテゴリの保存・表示は、集計・ルール評価・rollover・パスワード解錠・目標追跡に影響を与えてはならない（SHALL NOT）。既存の配分バー／today-group-breakdown のロジックも変更してはならない（SHALL NOT）。

#### Scenario: カテゴリ付与が集計を変えない
- **WHEN** タスクにカテゴリを付与する／外す
- **THEN** 総作業時間・ルール評価結果・目標レポート・解錠状態はいずれも変化しない
