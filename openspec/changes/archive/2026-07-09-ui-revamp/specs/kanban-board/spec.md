## ADDED Requirements

### Requirement: Cadence Board 風の 4 列カンバン

システムはカンバンを **保留 / 未着手 / 進行中 / 完了** の 4 列で表示 SHALL。各カードは優先度（高/中/低）・期限ラベル・タイトルを表示 SHALL。振り返りとはタブを分離 SHALL。

#### Scenario: 4 列とカード表示

- **WHEN** ユーザーがカンバンタブを開く
- **THEN** 保留・未着手・進行中・完了の 4 列が表示される
- **AND** 各カードに優先度バッジと期限ラベルとタイトルが表示される
- **AND** 振り返りは同じタブに同居しない

### Requirement: ドラッグ&ドロップでの列移動と完了

システムはカードのドラッグ&ドロップで列間移動を可能に SHALL。完了列へのドロップでタスクを完了扱いにし、完了演出の後にアーカイブ相当（当日ログへ記録）とする SHALL。

#### Scenario: 列間でカードを移動

- **WHEN** ユーザーがカードを別の列へドロップする
- **THEN** タスクの状態が移動先の列に更新される

#### Scenario: 完了列へドロップ

- **WHEN** ユーザーがカードを完了列へドロップする
- **THEN** タスクが完了として記録される
- **AND** 完了演出が表示され、当日アクティビティログに追加される

### Requirement: カードの優先度・期限・Markdown ノート

システムはカードに優先度（high/mid/low）・期限（date, 任意）・Markdown ノートを保持・編集可能に SHALL。

#### Scenario: カード詳細を編集

- **WHEN** ユーザーがカードを開き優先度・期限・ノートを編集して保存する
- **THEN** それらが永続化され、次回表示に反映される

### Requirement: 当日進捗ドーナツとアクティビティログ

システムは当日の完了/総数を示す進捗ドーナツと、当日完了タスクのアクティビティログを表示 SHALL。

#### Scenario: 進捗とログを表示

- **WHEN** ユーザーがカンバンタブを開く
- **THEN** 当日の達成率を示す進捗ドーナツが表示される
- **AND** 当日に完了したタスクのログが新しい順に表示される

### Requirement: Cadence Board の視覚仕様への忠実性

システムはカンバン画面の見た目を参照実装 `ref/kanban/Cadence Board.dc.html` に忠実に再現 SHALL。「風」ではなく、以下の検証可能な視覚要件を満たす SHALL（フォントのみ CSP 制約により system-ui フォールバック可）。

#### Scenario: キャンバスと列の見た目

- **WHEN** ユーザーがカンバンタブを開く
- **THEN** 画面キャンバスはクリーム色 `#F6F5F2`、基本文字色は `#37352F` である
- **AND** 各列は幅 290px・背景 `#FBFBF9`・枠 `#ECEBE7`・角丸 13px のカードで表示される
- **AND** 列ヘッダは「ひし形ドット（45度回転の 9px 正方形。完了=`#2E9E63`、進行中=`#4F86C7`、他=`#C7C5BE`）＋列名＋件数チップ」で構成される
- **AND** 件数チップは列ごとに配色される（完了=`#E9F5EE`/`#2E7D54`、進行中=`#E7F0FA`/`#2C5C8C`、他=`#F0EFEB`/`#8A8983`）

#### Scenario: カードの見た目

- **WHEN** タスクカードが表示される
- **THEN** カードは白背景・枠 `#EAE9E4`・角丸 10px・padding 12px で、ホバー時に浮き上がる（影強調＋1px 上移動）
- **AND** 上段に優先度チップ（高=`#FBEAE6`/`#B4493B`、中=`#FAF1DD`/`#9A7A1F`、低=`#EEF1F4`/`#5E7E96`）と期限ラベル（`#9B9A94`、「今日/明日/昨日/M/D/M/D 超過/期限なし」形式）が並ぶ
- **AND** タイトルは 14.5px / weight 600 で表示される

#### Scenario: 列内インラインのタスク追加

- **WHEN** ユーザーが列末尾の「＋ 新規タスク」を押す
- **THEN** その列の中にインラインのテキスト入力（コンポーザ）が開く
- **AND** Enter で追加して入力を継続、Esc で取消、フォーカス喪失で確定して閉じる
- **AND** 完了列には「＋ 新規タスク」を表示しない
- **AND** ボード上部にツールバー型の追加フォームを**置かない**

#### Scenario: 完了列とアーカイブ

- **WHEN** 完了列が空である
- **THEN** 破線ボックスで「ここにドロップでタスク完了（完了後は自動でアーカイブ）」の案内を表示する
- **WHEN** カードを完了列へドロップする
- **THEN** カード上にチェック演出（フロスト＋チェックマーク）と粒子の祝福演出が再生され、約 1 秒後にカードはボードから消えてアクティビティログと完了件数に反映される

#### Scenario: サイドバー（進捗・ログ）

- **WHEN** カンバンタブを開く（詳細パネル非表示時）
- **THEN** 右サイドに「本日の進捗」カード（SVG ドーナツ 140px・緑 `#2E9E63`・中央に完了数、右に達成率%・期限超過件数・アーカイブ件数）と「アクティビティログ」カード（緑のチェック丸＋「『タイトル』を完了しました」＋時刻）が表示される

#### Scenario: 詳細パネル

- **WHEN** カードをクリックする
- **THEN** 右サイドが詳細パネルにスライドインで切り替わる（モーダルは使わない）
- **AND** パネルは「閉じるボタン／タイトル／優先度ピル選択（高・中・低）／期限のカレンダーポップオーバー（月送り・今日・明日・期限なし）／Markdown ライブエディタ」で構成される
- **AND** ライブエディタは行単位のブロック編集（見出し・チェックボックス・箇条書き・番号リスト・引用、`#` `[ ]` `-` `1.` `>` ショートカット、インラインの `code`/**太字**/*斜体*）に対応する

### Requirement: PLANNING シグナルの契約を維持

システムはゲートの PLANNING 判定（翌日タスク登録）を維持 SHALL。カンバンで翌日を期限とする（または翌日に予定された）タスクを 1 つ以上登録すると、PLANNING の翌日タスク要件を満たせる SHALL。

#### Scenario: 翌日タスク登録で PLANNING を満たす

- **WHEN** ユーザーが期限（または予定日）を翌日とするタスクを登録する
- **THEN** ゲートの PLANNING 評価における翌日タスク数が増加する
- **AND** 既存の PLANNING/振り返り条件の評価は従来どおり動作する
