# Proposal: goal-30day-challenge

## Why

日次ルールとゲート（issue #9）は「今日を守らせる」ことはできるが、すべてが日単位でぶつ切りのため、1ヶ月単位の取り組みを積み上げとして見る手段がない。欲しいのは「30日間、思いつきであっても無心で取り組み続け、30日後に成果を見て『あ、こんなに進んだんだ』と思える」こと。設計議論（issue #9 / `ref/goal-report/design-brief.md`）で、全部盛りではなくミニマム構成（ヘッダ＋4ブロックのレポート）から始める方針が確定した。

## What Changes

- **30日チャレンジ（目標）エンティティを新設**する。目標名・目的の一文・開始日からの30日固定期間を持つ。並行して複数走らせられる。成否ラベルは存在せず、30日経過＝「完走」だけがある。
- **既存のルール条件を「実践」として採用（adopt）**する。採用は安定した同一性（`total_work` / `group:<stableGroupId>` / `planning:<signalKey>`）で行い、日次評価の仕組み（`unlock_evaluation`・当日凍結・rollover）には一切手を入れない。
- **期間中のジャンル固定・強度可変**: 目標が採用中の条件は、未来ルール編集で削除・対象変更できない。閾値（`threshold_seconds`）の変更は許可するが、**変更理由の一言を必須**とし記録する。当日凍結は既存機構のまま（当日の泣きの一手は不可、変更は翌日から）。
- **作成当日中のみ目標を無条件削除できる**（誤作成の救済）。翌日以降は削除・放棄の手段を提供しない。
- **目標日記**: 振り返りタブに、アクティブな目標ごとの日記コーナーを設ける。本文は `reflection_entry` とは別に保存し、`reflection_done` シグナル（振り返り本文の非空判定）を汚染しない。
- **完了レポート画面**: 30日経過した目標に対し、ヘッダ（目標名・目的・期間・達成 N/30）＋4ブロック（①実践別の30日達成カレンダー ②時間型実践の実測と閾値の推移＋理由マーカー ③Day1/Day30 の文面 Before/After ④日記の1件読みリーダー）を表示する。既存の永続データ（`unlock_evaluation.per_condition_results`・`daily_totals_snapshot`・過去ルールセット）から集計する。
- **スコープ外（将来の change）**: 画像添付基盤（Before/After 画像・証拠写真）、タイムライン手動記録による判定（TIMELINE 条件タイプ）、MANUAL_CHECK の採用、Day0 儀式の拡張（手紙・予想・ベースライン snapshot）、期間中の演出（ペース予測・封印）、候補カード群（解錠時刻推移・相関・換算・ナラティブ・トロフィー棚）。

## Capabilities

### New Capabilities

- `goal-challenge`: 30日チャレンジのライフサイクル。作成（名前・目的・実践の採用）、並行運用、期間中のジャンル固定と理由付き閾値変更、作成当日限りの削除猶予、完走判定（=期間満了のみ）。
- `goal-journal`: 目標ごとの日記。振り返りタブのコーナーでの記入・保存と、`reflection_entry` から独立した永続化。
- `goal-report`: 完走した目標の完了レポート。ヘッダ＋4ブロック構成と、既存永続データからの集計規則。

### Modified Capabilities

（なし — 既存 spec の要求は変更しない。ルール編集のバリデーション強化・振り返りタブへのコーナー追加はいずれも新 capability 側の要求として記述する）

## Impact

- **DB**: migration v11 で新テーブル（目標・採用実践・閾値変更ログ・目標日記）を追加。既存テーブルは無変更。
- **server**: `rules.ts` の `upsertFutureRuleSet` / `deleteRuleSet` に採用条件のバリデーションを追加。`services/goals.ts`（新規）と API エンドポイント群。集計・評価・rollover（`aggregation/` / `evaluate.ts` / `rollover.ts`）は無改造。
- **dashboard**: 新しい「目標」タブ（一覧・作成・レポート）、振り返りタブへの日記コーナー追加。
- **参照**: GitHub issue #9、デザインブリーフ `ref/goal-report/design-brief.md`（レポートの視覚方針）。
