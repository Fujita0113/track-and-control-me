## ADDED Requirements

### Requirement: 編集の取り消し・やり直し（Undo/Redo）

インライン・ライブ Markdown エディタは、DOM 丸ごと再構築で失われるブラウザ標準の Undo 履歴を、raw テキスト + キャレット位置のスナップショットによる**自前の履歴スタック**で置き換えなければならない（MUST）。ユーザーは `Ctrl+Z` で直前の編集を取り消し、`Ctrl+Shift+Z` または `Ctrl+Y` でやり直せなければならない（MUST）。取り消し/やり直しの復元はキャレット位置も含めて行う（MUST）。

連続した文字入力は 1 履歴にコアレス（合体）し、1 キーストローク＝1 履歴にしてはならない（MUST NOT）。Enter による行操作・貼り付け・切り取り・チェックボックストグル・行移動・インデント変更・装飾ラップ等の構造変化はコミット境界として個別の履歴に区切らなければならない（MUST）。IME 変換中（composition 中）はスナップショットを取らず、確定後の状態を履歴に記録する（MUST）。

#### Scenario: Ctrl+Z で直前の入力を取り消す
- **WHEN** ユーザーが本文を入力した後に `Ctrl+Z` を押す
- **THEN** 直前のコミット境界の raw テキストへ戻り、キャレットもその時点の位置に復元され、ブラウザ標準の undo によるページ全体の巻き戻りは起きない

#### Scenario: Ctrl+Shift+Z / Ctrl+Y でやり直す
- **WHEN** 取り消した直後に `Ctrl+Shift+Z`（または `Ctrl+Y`）を押す
- **THEN** 取り消した編集が再適用され、キャレットもやり直し後の位置に復元される

#### Scenario: 連続入力を 1 回で取り消す
- **WHEN** ユーザーが数文字を連続してタイプし、その後 `Ctrl+Z` を 1 回押す
- **THEN** 直前の 1 文字だけでなく、コアレスされた連続入力のまとまりが一度に取り消される

#### Scenario: 取り消し後に新規入力すると redo 分岐は破棄される
- **WHEN** 取り消した状態から新しく文字を入力する
- **THEN** それ以降に `Ctrl+Y`／`Ctrl+Shift+Z` を押しても取り消し済みだった内容は復元されない（redo スタックは破棄される）

### Requirement: プレーンテキストとしての Markdown 貼り付け

エディタへの貼り付け（paste）は既定のリッチ貼り付けを抑止し、クリップボードの `text/plain` を取り込んで「1 行 = 1 ブロック div」モデルへ反映しなければならない（MUST）。貼り付けはキャレット位置（選択があればそれを置換して）に挿入し、改行を正規化（`\r\n?` → `\n`）した上で raw テキストへ差し込み、再描画とキャレット再設定を行う（MUST）。`<div>`・`<br>`・スタイル付き span 等の HTML 構造を取り込んではならない（MUST NOT）。貼り付けは 1 つのコミット境界として Undo 履歴に記録しなければならない（MUST）。

#### Scenario: 複数行テキストが行ごとのブロックになる
- **WHEN** ユーザーが改行を含む複数行のテキスト（例: 4 行の箇条書き）を空の位置に貼り付ける
- **THEN** 各行がそれぞれ独立した「1 行 = 1 ブロック」として描画され、複数行が 1 かたまりに潰れない

#### Scenario: Markdown 記法が貼り付け後にライブ整形される
- **WHEN** `- ` や `# ` などの Markdown 記法を含むテキストを貼り付ける
- **THEN** 取り込まれた raw テキストが再描画され、箇条書き・見出し等がインライン・ライブ整形で表示される（貼り付け元の書式ではなく本エディタの整形結果になる）

#### Scenario: 行の途中に貼り付ける
- **WHEN** ある行の途中にキャレットを置いて複数行テキストを貼り付ける
- **THEN** キャレット前後のテキストは保持されたまま貼り付け内容が差し込まれ、貼り付け直後の位置にキャレットが置かれる

#### Scenario: 貼り付けを Ctrl+Z で取り消す
- **WHEN** 貼り付けを行った直後に `Ctrl+Z` を押す
- **THEN** 貼り付け前の raw テキストとキャレット位置に一度で戻る

### Requirement: Shift+Enter による素の改行（ブロックモデルを壊さない）

`Shift+Enter` は、マーカーを継続しない**素の改行**を raw モデル上に挿入しなければならない（MUST）。すなわち Enter ハンドラは `Shift` の有無に関わらず改行を自前で処理し、`Shift` あり時はリスト/引用マーカーの継続を行わず、現在行を分割した新しい段落行を作る（MUST）。ブラウザ既定に落ちて `contenteditable` 内に `<br>` を挿入させてはならない（MUST NOT）——`<br>` は `textContent` 連結で改行として復元されず、行融合による内容欠損を招くため。

#### Scenario: Shift+Enter で行が融合しない
- **WHEN** ユーザーがリスト項目・引用・段落の行末または途中で `Shift+Enter` を押す
- **THEN** マーカーを継続しない新しい行が挿入され、「1 行 = 1 ブロック」モデルが保たれ、直後の再描画で 2 行が 1 行に融合したりキャレットが飛んだりしない

#### Scenario: Enter との役割分担
- **WHEN** リスト項目の行末で `Enter`（Shift なし）を押す
- **THEN** 従来どおり同じマーカーで継続する（Shift+Enter はマーカーを外した素の行になる、という違いが一貫して成立する）

### Requirement: 行の入れ替え（Alt+↑ / Alt+↓）

`Alt+↑`（`Alt+↓`）で、キャレットのある行を直上（直下）の行と入れ替えられなければならない（MUST）。入れ替えはブラウザ既定のキャレット移動を抑止し、raw 上で行を swap して再描画・キャレット再設定する（MUST）。文書の先頭行で `Alt+↑`、末尾行で `Alt+↓` を押した場合は何もしない（MUST）。行移動は 1 つのコミット境界として Undo できる（MUST）。

#### Scenario: 行を上下に並べ替える
- **WHEN** ユーザーがある行にキャレットを置いて `Alt+↑`（または `Alt+↓`）を押す
- **THEN** その行が直上（直下）の行と入れ替わり、キャレットは移動先の行の対応桁（行長でクランプ）に置かれる

#### Scenario: 文書端では何もしない
- **WHEN** 先頭行で `Alt+↑`、または末尾行で `Alt+↓` を押す
- **THEN** 行順は変わらず、キャレット位置も保たれる

### Requirement: リストのインデント / アウトデント（Tab / Shift+Tab）

リスト項目・タスク項目上での `Tab`（`Shift+Tab`）は、行頭のインデントを 1 段（2 スペース）増やす（減らす）ことでネストを操作できなければならない（MUST）。`Tab` はブラウザ既定のフォーカス移動を抑止し（`preventDefault`）、エディタ内に留まらなければならない（MUST）。インデントは raw テキスト中の実際の半角スペースとして保持し、描画側のネスト段差（`indent.length` 由来のインデント幅）に反映される（MUST）。インデント変更は 1 つのコミット境界として Undo できる（MUST）。

#### Scenario: 箇条書きを一段ネストする
- **WHEN** ユーザーがリスト/タスク項目上で `Tab` を押す
- **THEN** 行頭に 2 スペースのインデントが加わり、描画がネスト段差で表示され、キャレット桁は +2 されてフォーカスはエディタに留まる

#### Scenario: アウトデントは先頭で止まる
- **WHEN** インデントのないリスト項目で `Shift+Tab` を押す
- **THEN** インデントはこれ以上減らず（負にならず）、内容も壊れない

### Requirement: 選択テキストの装飾ラップ（Ctrl+B / Ctrl+I / Ctrl+E）

`Ctrl/Cmd+B`・`Ctrl/Cmd+I`・`Ctrl/Cmd+E` は、選択テキストをそれぞれ `**`（太字）・`*`（斜体）・`` ` ``（インラインコード）で囲まなければならない（MUST）。既に同じマーカーで囲まれている場合はトグルで解除する（MUST）。ブラウザ既定の `execCommand('bold')` 等による `<b>`/`<i>`/font span の注入を抑止しなければならない（MUST NOT）。ラップ後は対象テキストの選択を保持し、続けて再トグルできる（MUST）。

#### Scenario: 選択を太字にする / 解除する
- **WHEN** ユーザーがテキストを選択して `Ctrl+B` を押す
- **THEN** 選択が `**…**` で囲まれ、`<b>` 等の HTML は注入されず、ラップ後もその範囲が選択されたままになる
- **WHEN** 既に `**…**` で囲まれた範囲で再度 `Ctrl+B` を押す
- **THEN** マーカーが取り除かれて素のテキストに戻る

#### Scenario: 選択が無いときは空ペアを挿入
- **WHEN** 選択が無い（キャレットのみ）状態で `Ctrl+I` を押す
- **THEN** 空の `**`/`*`/`` ` `` ペアが挿入され、キャレットがマーカーの内側に置かれる

### Requirement: キーボードによるチェックボックス切替（Ctrl/Cmd+Enter）

キャレットがタスク（チェックボックス）行にあるとき、`Ctrl/Cmd+Enter` で `[ ]`⇄`[x]` をトグルできなければならない（MUST）。トグルは長さ不変の置換でキャレット位置を保持する（MUST）。非タスク行では改行を挿入せず、そのキー入力を消費する（no-op）（MUST）。トグルは 1 つのコミット境界として Undo できる（MUST）。

#### Scenario: タスク行でチェックを切り替える
- **WHEN** キャレットがタスク行にある状態で `Ctrl+Enter`（または `Cmd+Enter`）を押す
- **THEN** その行の `[ ]` と `[x]` がトグルされ、キャレットは同じ位置に留まり、改行は挿入されない

#### Scenario: 非タスク行では何もしない
- **WHEN** タスクでない行で `Ctrl+Enter` を押す
- **THEN** 改行は挿入されず内容も変化しない

### Requirement: モデル経由の切り取り（cut）

切り取り（`cut`）は、選択範囲の raw テキストをクリップボードへ `text/plain` として書き出し、raw モデルから当該範囲を除去しなければならない（MUST）。ブラウザ既定の `cut` によるブロック跨ぎの DOM 破壊に委ねてはならない（MUST NOT）。切り取り後はキャレットを除去位置に置き、1 つのコミット境界として Undo できる（MUST）。選択が無い場合は何もしない（MUST）。

#### Scenario: 複数ブロックにまたがる選択を切り取る
- **WHEN** ユーザーが複数行にまたがるテキストを選択して切り取る
- **THEN** 選択部分の raw がクリップボードに入り、raw から取り除かれ、残りの行構造が壊れず、`Ctrl+Z` で 1 回で元に戻せる

### Requirement: IME 変換中の getValue 整合（保存でデータを落とさない）

`getValue()` は IME 変換中（composition 中）でも、DOM 上の未確定テキストを含む現在の本文を返さなければならない（MUST）。すなわち変換中は内部 `raw`（composition ガード付きパスでしか更新されない）ではなく `getContent(editor)` を読み、確定前の保存・フラッシュで最終行の未確定文字が欠落してはならない（MUST NOT）。

#### Scenario: 変換確定前に保存しても文字が消えない
- **WHEN** ユーザーが最終行で日本語を変換中（未確定）に、保存・日付切替・タブ離脱などで本文が読み出される
- **THEN** 未確定テキストを含む現在の内容が返り、保存結果から最終行の文字が欠落しない

### Requirement: 構造付き HTML 貼り付けの Markdown 化

エディタへの貼り付けは、クリップボードに構造付き `text/html`（リスト・見出し・引用・チェックリスト等）が含まれる場合、それを解析して「1 行 = 1 ブロック」の raw Markdown へ変換し取り込まなければならない（MUST）。`<ul>`/`<ol>` の各 `<li>` は箇条書き（`- `）/番号（`N. `）、ネストは 1 段あたり 2 スペースのインデント、チェックリスト項目は `- [ ] `/`- [x] `（チェック状態を保持）、`<h1>`–`<h6>` は `#` の数、`<blockquote>` は各行 `> ` に対応づける（MUST）。変換はタグ駆動・クラス非依存とし、Notion・Google Docs・一般的なセマンティック HTML を単一経路で扱う（MUST）。解析は `DOMParser` の detached document を読み取るのみとし、クリップボード HTML を live エディタへ注入してはならない（MUST NOT）。構造タグを含まない HTML（inline のみ）の場合は `text/plain` にフォールバックして改行を保持する（MUST）。貼り付けは 1 つのコミット境界として Undo できる（MUST）。

#### Scenario: 箇条書きが行ごとのバレットになる
- **WHEN** 箇条書き（`text/html` に `<ul><li>` を含む）をコピーして貼り付ける
- **THEN** 各項目が `- ` のバレット行として取り込まれ、複数行が 1 かたまりに潰れず、ライブ整形で箇条書き表示される

#### Scenario: チェックリストがタスク行になる
- **WHEN** チェックボックス項目（Notion の to-do リスト、または GFM の `<input type="checkbox">`）を貼り付ける
- **THEN** 各項目が `- [ ] ` / `- [x] `（チェック状態を保持）として取り込まれ、チェックボックスとして描画される

#### Scenario: ネスト・見出し・引用が対応する Markdown になる
- **WHEN** ネストした箇条書き・見出し・引用を含む HTML を貼り付ける
- **THEN** ネストは 2 スペース/段のインデント、見出しは `#` の数、引用は `> ` の raw Markdown に変換される

#### Scenario: 構造の無い貼り付けは改行を保持する
- **WHEN** 構造タグを含まない HTML（inline のみ）と改行付き `text/plain` を持つ内容を貼り付ける
- **THEN** `text/plain` の改行が保持され、1 かたまりに潰れない

### Requirement: タスクチェックボックスの Notion 風表示

タスク行のチェックボックスは、raw の先頭リストマーカー（`- ` 等）と `[ ]`/`[x]` の文字を DOM の textContent に保持したまま、視覚的には Notion 風の角丸チェックボックスとして描画しなければならない（MUST）。未チェックは枠線のみ、チェック済みは塗り＋チェックマークで表し、チェック済み行の本文は取り消し線とミュート表示にする（MUST）。文字の視覚的な非表示は CSS のみで行い、raw モデル（textContent からの raw 復元・グローバル文字オフセット・`TASK_RE` の一致）に影響を与えてはならない（MUST NOT）。追加アセット（画像 URL 等）に依存せず CSP に適合する（MUST）。

#### Scenario: チェックボックスが角括弧でなくボックスで表示される
- **WHEN** タスク行 `- [ ] xxx` / `- [x] xxx` が描画される
- **THEN** 生の `[ ]`/`[x]` や先頭 `- ` は表示されず、角丸のチェックボックス（未チェック=枠線／チェック=塗り＋チェックマーク）と本文が表示され、チェック済み行は取り消し線＋ミュートになる

#### Scenario: 本文中の装飾マーカーは隠れない
- **WHEN** タスク本文に `**太字**` 等のインライン装飾が含まれる
- **THEN** 先頭のリストマーカーだけが視覚的に隠され、本文中の装飾マーカーは従来どおり表示される（子結合子でスコープされる）
