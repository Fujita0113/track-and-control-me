## Context

かんばん（`server/static/js/kanban.js`）では、完了列（`DONE`）にドロップしたタスクを `completeTask` が `status='DONE'` / `done_at=now` に更新し、演出後にボードから消す。以降そのタスクは右サイドの `logEl`（アクティビティログ）に「本日完了分」としてのみ現れる。

- 完了判定は `normStatus(t.status) === 'DONE'`、当日フィルタは `localDateKey(done_at) === state.today`。
- サーバの PATCH `/api/tasks/:id` は任意の `status` を受け付け、`updateTask`（`server/src/services/tasks.ts`）は「`DONE` 以外へ遷移したら `done_at = NULL`」を既に実装済み。
- 派生表示（`completedTodayCount` / `archivedCount` / 達成率 / ドーナツ）はすべて `S.tasks` から算出され、`renderAll()` で再描画される。

したがって「復帰」に必要な状態遷移はサーバ側に既に存在し、本変更は純粋にフロントの操作導線と再描画の追加で完結する。

## Goals / Non-Goals

**Goals:**
- アクティビティログの各行から 1 クリックで、誤ってアーカイブしたタスクをボードへ戻せるようにする。
- 復帰後に完了件数・アーカイブ件数・達成率・ドーナツ・列カードが即座に整合する。
- サーバ／DB／API の変更なしで実現する（既存 PATCH の再利用）。

**Non-Goals:**
- 元の列（完了前に居た列）への正確な復元。復帰前の列は保持していないため対象外。
- 過去日（本日以外）に完了したタスクの復帰。ログは当日分のみ表示するため導線も当日分に限る。
- 完了→アーカイブの演出・挙動そのものの変更。

## Decisions

### D1: 復帰先の列は未着手（TODO）固定
タスク行は「完了前にどの列に居たか」を保持しないため、正確な復元はできない。取り得る選択肢は (a) `TODO` 固定、(b) `DOING` 固定、(c) 復帰前の列を新たに記録して復元。
- (c) は DB 列追加＋保存経路の変更を伴い、「サーバ変更なし」という本変更の軽量性を損なう。
- 誤アーカイブの取り消しは「もう一度トリアージし直す」に等しく、中立な入口である `TODO`（未着手）が最も自然。
→ **`TODO` 固定**を採用。`DOING` は「作業中」という状態を勝手に主張してしまうため退ける。

### D2: 既存 `updateTask` PATCH を再利用（サーバ変更ゼロ）
`api.updateTask(id, { status: 'TODO' })` を呼ぶだけで、サーバ側 `updateTask` が `done_at = NULL` を刻む。復帰専用エンドポイントは作らない。楽観的更新として、先にローカル `S.tasks` の当該タスクを `status='TODO' / done_at=null` にし列末尾へ挿入 → `renderAll()` → PATCH の順で反映する。

### D3: 楽観更新＋失敗ロールバック
`completeTask` は失敗時に `toast` するのみだが、復帰は「消えていたカードが再表示される」ため、失敗を放置するとローカルとサーバが乖離する。PATCH 失敗時は当該タスクを復帰前（`DONE` / 元 `done_at`）へ戻し `renderAll()`、`toast(..., 'err')` する。実装簡素化のため、失敗時は `reload()`（サーバ状態の再取得）でローカルを真実に一致させる方式も可とする。

### D4: 導線は「戻す」ボタンをログ行に付与
`logEl` の各行（`kb-log-row`）末尾に小さな「戻す」ボタンを追加する。完了演出との視覚的整合のため控えめなテキスト／アイコンボタンとし、クリックで `restoreTask(t)` を呼ぶ。行クリック等の既存挙動が無いことを確認済み（ログ行は現状インタラクション無し）。

## Risks / Trade-offs

- [復帰先が元の列と異なる（常に TODO）] → 誤アーカイブの取り消しという主目的では許容範囲。ユーザーは復帰後にドラッグで再配置できる。設計判断として proposal/spec に明記。
- [楽観更新でカードが一瞬出てから消える（失敗時）] → 失敗時ロールバックで整合。頻度は低い（ローカルサーバ前提）。
- [当日分ログのみが対象] → 日付跨ぎで完了したタスクは復帰導線に出ない。Non-Goal として明示。

## Migration Plan

- スキーマ変更なし・後方互換。フロントのビルド／配信（`server/static`）のみで反映。ロールバックはコード revert で完結。
