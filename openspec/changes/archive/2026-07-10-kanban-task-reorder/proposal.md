## Why

カンバンでは現在カードを列間でしか動かせず（＝状態変更のみ）、列内での並び順は変更できない。全カードの `sort_order` が `0` のまま作成されるため、列内順序は事実上「作成順（id 昇順）」に固定されている。ユーザーは「未着手」列などの中でタスクの順番を自分で整えたい（優先して取り組む順に並べ替えたい）ため、列内の並べ替え手段が必要。

## What Changes

- カードを**同一列内でドラッグ&ドロップして並べ替え**できるようにする。ドロップ位置に応じて挿入先インデックスを計算し、その順序を永続化する。
- 列内での並び順を `sort_order`（既存の DB 列・PATCH 対応済み）で明示的に管理する。ドラッグ中は挿入位置を示すプレースホルダ（インジケータ）を表示する。
- **列間移動時**も、末尾固定ではなくドロップ位置に応じたインデックスへ挿入されるようにする（現在は状態のみ更新し位置は成り行き）。
- **新規カード作成時**に明示的な `sort_order`（当該列の末尾ランク）を付与し、全カードが `0` で衝突する現状を解消する。
- 既存カード（`sort_order = 0` で並んでいる）を初回操作時に列単位で正規化（0,1,2,… へ振り直し）する。
- 完了列（DONE）は完了演出＋アーカイブ専用のため、並べ替え対象外とする（従来どおり）。
- ドラッグ操作ヒント文言（「列の移動はボードでカードをドラッグ」）を並べ替えにも触れる形へ更新する。

破壊的変更なし（`sort_order` は既存スキーマ・既存 API で対応済み。データ移行不要）。

## Capabilities

### New Capabilities
- `kanban-task-reorder`: カンバンの列内でカードをドラッグ&ドロップして並び替え、その順序を `sort_order` で永続化する挙動。挿入位置インジケータ、新規カードへの末尾ランク付与、列単位の順序正規化、列間移動時の位置指定を含む。

## Impact

- **フロントエンド** `server/static/js/kanban.js`: ドラッグ&ドロップのメインロジック。`onDrop` の「同一列は早期リターン」分岐を挿入インデックス計算へ置き換え、`dragover` に列全体ハイライトではなく位置ごとのヒットテスト＋挿入プレースホルダを追加。新規カード作成（`createTask` 呼び出し）に `sort_order` を付与。列内正規化ヘルパを追加。ヒント文言更新。
- **フロントエンド** `server/static/js/api.js`: 複数カードの並び替えを 1 リクエストで保存する場合はバッチ用メソッドを追加（単発なら既存 `updateTask(id, { sort_order })` を流用可）。
- **バックエンド** `server/src/api/planning.ts` / `server/src/services/tasks.ts`: 列内一括再インデックスを atomic に行うためのバッチ `reorder` エンドポイント／サービス関数を追加（任意。`sort_order` の PATCH 自体は既に対応）。PATCH ごとに planning パイプラインが再実行される点に留意。
- **DB** `server/src/db/migrations.ts`: `task(status, sort_order)` の索引追加は任意（並べ替えの正しさには不要、読み取り最適化のみ）。
- **参照実装** `ref/kanban/Cadence Board.dc.html`: 参照実装にも列内並べ替えは無いため流用不可。本機能は参照実装に無い新規挙動として設計する（視覚トーンは Cadence Board に合わせる）。
