## ADDED Requirements

### Requirement: アクティブタブのグループ検出

システムの拡張機能（Chromium/Edge, Manifest V3）は、**現在フォーカスされているアクティブタブが属するタブグループ**を検出し、その識別子・グループ名・タイムスタンプを含むイベントをローカルアプリへ送信 SHALL する。
グループに属さないタブがアクティブな場合は、予約バケット `ungrouped`（"その他"）として扱う SHALL。
また拡張機能は、同時オープンの分配（divide-by-N）に用いるため、**その時点で開いているタブグループの集合（openGroupKeys）**も併せて送信 SHALL する。

#### Scenario: グループ内タブがアクティブになる
- **WHEN** ユーザーがグループ "AtCoder" に属するタブをフォーカスした
- **THEN** 拡張機能は `{ groupId, groupTitle: "AtCoder", browserFocused: true, idleState: "active" }` を含むイベントをローカルアプリへ送信する

#### Scenario: 未グループタブがアクティブになる
- **WHEN** ユーザーがどのグループにも属さないタブをフォーカスした
- **THEN** 拡張機能はグループを `ungrouped` として送信し、バックエンドは "その他" バケットに時間を計上する

#### Scenario: グループを閉じると分配対象が減る
- **WHEN** それまで "AtCoder" と "開発" の2グループが開いていたが、"AtCoder" のグループを閉じた
- **THEN** 以降の時間は残った "開発"（分母1）に全額計上される

#### Scenario: 同時に開いているグループ集合の報告
- **WHEN** グループ "AtCoder" と "課題" の両方が開いている状態でイベントが送信される
- **THEN** イベントには現在フォーカス中のグループに加え、開いているグループ集合（openGroupKeys: AtCoder, 課題）が含まれる

### Requirement: グループ別アクティブ秒数の日次集計

バックエンドは、受信したイベント列から**タブグループごとのアクティブ秒数**を集計・永続化 SHALL する。集計は**設定された日境界（既定: 午前4時）で区切られる「1日」単位**で行い、アプリ再起動をまたいでも失われない SHALL。
グループの同一性は、再起動で振り直される Edge の groupId ではなく、**安定キー（stableGroupId。フォールバックはグループ名＋色）**で判定 SHALL する。

#### Scenario: 単一グループの継続利用
- **WHEN** ユーザーがグループ "AtCoder" を 25 分間アクティブに使い続けた
- **THEN** その「1日」の "AtCoder" のアクティブ秒数が約 1500 秒（±ハートビート間隔）増加する

#### Scenario: 午前4時をまたぐセッションの日帰属
- **WHEN** 深夜 03:30 から 05:00 まで同一グループを継続利用した
- **THEN** 03:30〜04:00 の分は前日に、04:00〜05:00 の分は当日に帰属して集計される

#### Scenario: ブラウザ再起動をまたいだ同一グループ
- **WHEN** ブラウザを再起動し、同名・同色のグループ "AtCoder" を再び利用した
- **THEN** 再起動前後の時間は同一の安定キーで合算され、別グループに分裂しない

#### Scenario: アプリ再起動後も集計が保持される
- **WHEN** バックエンドを再起動した
- **THEN** 再起動前までに集計済みのグループ別秒数は保持され、以降の計上が継続する

### Requirement: スリープ・アイドル・ロック中の非カウント

システムは、以下のいずれかの区間を**アクティブ時間として計上しない** SHALL:
(a) `chrome.idle` の状態が `idle` または `locked`、
(b) ハートビートが一定閾値（既定 90 秒）を超えて途切れている（＝OSスリープ／Edge 終了とみなす）。
一方で、**Edge が最前面でない（他アプリで作業中）だけでは計上を止めない** SHALL — 在席（`active`）かつ Edge 起動中であれば、開いているグループへ計上する。

#### Scenario: アイドル状態
- **WHEN** ユーザー無操作が続き `chrome.idle` が `idle` に遷移した
- **THEN** `idle` 遷移以降の時間はどのグループにも計上されない

#### Scenario: 画面ロック
- **WHEN** OS 画面がロックされ `chrome.idle` が `locked` になった
- **THEN** ロック中の時間は計上されない

#### Scenario: OSスリープによるハートビート欠落
- **WHEN** PC がスリープし、ハートビートが閾値（例: ハートビート間隔の数倍）を超えて途切れた
- **THEN** 欠落区間はアクティブ時間として計上されない（ギャップは上限でクランプされる）

#### Scenario: Edge が最前面でなくても在席なら計上
- **WHEN** ユーザーが VS Code で作業中（Edge は背面）だが Edge に「開発」グループを開いたままで、`chrome.idle` は `active`
- **THEN** その時間は「開発」グループ（他に開いているグループがあれば分配して）に計上される

### Requirement: 同時オープン・グループへの時間の均等分配（divide-by-N）

システムは、ある区間に**複数のタブグループが同時に開いている**場合、その区間の時間を**開いている全グループへ均等に分配（各 1/N）** SHALL する。分配後の合計はその区間の実時間を超えない SHALL（二重計上ではなく分配）。開いているグループが1つだけの区間は、その全時間を当該グループへ計上 SHALL する。この分配は Edge が最前面か否かに依らず、在席（`active`）かつ Edge 起動中の区間に適用 SHALL する。

#### Scenario: 2グループ同時オープンの均等分配
- **WHEN** グループ "AtCoder" と "開発" が同時に開いた状態が 2 時間続いた（在席）
- **THEN** それぞれに 1 時間ずつ計上され、合計は実時間の 2 時間を超えない

#### Scenario: 単一グループのみ
- **WHEN** 開いているグループが "開発" 1 つだけの区間が 30 分続いた
- **THEN** その 30 分は全て "開発" に計上される

#### Scenario: 別アプリ作業中も開いているグループへ計上
- **WHEN** ユーザーが VS Code でコーディング中（Edge は背面）で、Edge には "開発" グループのみ開いている
- **THEN** その時間は "開発" に計上される（VS Code 自体は追跡しない）

#### Scenario: 誤って開いていたグループの手動除外
- **WHEN** ある区間で "AtCoder" と "資料" が開いていたが、実際は "資料" を使っていなかったため、行動記録画面で "資料" の割合を 0 にした
- **THEN** その区間の時間は "AtCoder" に再割当され、区間内の総和は実時間のまま保たれる

### Requirement: ローカル完結の通信

拡張機能とバックエンド間の通信は **localhost に限定** SHALL され、外部ネットワークへ活動データを送信しない SHALL。

#### Scenario: 外部送信の不在
- **WHEN** 拡張機能がイベントを送信する
- **THEN** 送信先は localhost 上のローカルアプリのみであり、外部ホストへの送信は発生しない
