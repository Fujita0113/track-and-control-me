## Why

開発者である自分自身の「総作業時間の底上げ」と「メンタルの安定」を、ゲームを報酬にした
**コミットメントデバイス**で実現する。ゲーム用PCのパスワードは毎日ロジックで自動変更されるため、
「あらかじめ決めた作業条件（例: 総作業4時間以上かつ競技プログラミング15分以上）を満たすまで
パスワードが分からない＝ゲームできない」状態を作る。作業の計測は、Edge のタブグループを
「活動の単位」とみなして自動で行い、手作業の記録負荷をゼロに近づける。

対象ユーザーは開発者本人のみ（シングルユーザー・ローカル完結）。

## What Changes

- **Edge タブグループ単位の作業時間計測**を新設する。Chromium 拡張機能（Manifest V3）が
  「今フォーカスされているタブのグループ」を検出し、ローカルアプリへ送信。バックエンドが
  グループごとの「アクティブ秒数」を日単位で集計する。
- **スリープ／アイドル／ロック中はカウントしない**。`chrome.idle` の状態と、ハートビートの
  欠落（＝OSスリープ）を突き合わせて非アクティブ区間を除外する。**同時に複数グループが開いている区間は、
  開いている全グループへ時間を均等分配（divide-by-N。合計は実時間を超えない）**。計上は Edge が最前面か
  否かに依らない（在席かつ Edge 起動中なら計上）ため、**「開発」グループを開いておけば VS Code 等での
  作業時間も自動で計上**できる（VS Code 自体は追跡しない）。放置・誤オープンは行動記録画面で割合を編集（0 可）して補正する。
- **活動カテゴリ**を導入する。各タブグループ（例: "AtCoder"）をカテゴリ（例: 競技プログラミング）へ
  マッピングし、「作業とみなすカテゴリ」の合計から「総作業時間」を算出する。
- **日次ルールエンジン**を新設する。カテゴリ別・総合の時間しきい値に加え、**手動チェック（チェックボックス）**や
  **「翌日計画完了」**などのブール条件でアンロック条件を定義。
  「1日」は **午前4時区切り**（0:00〜4:00は前日扱い）。**その日のルールは開始後は変更不可（凍結）**、
  **翌日以降のルールは変更可能**。一度達成した日の判定は解除されない（latch）。
  なお、条件未達を回避する**アプリ内の脱出弁（break-glass）は設けない**（＝強制力を担保する。
  唯一の逃げ道は「意図的に .ps1 を手実行」する手間のかかる行為のみ）。
- **パスワードゲート**を新設する。条件達成時に、既存のパスワード生成ロジック（日付を渡すと
  その日のパスワードを返す差し替え可能なコマンド）を呼び出し、**前日と当日の2つの候補**を表示する
  （ゲームPC側のタスクスケジューラが失敗して前日のパスワードのままになる事象への保険）。
- **振り返り＋翌日タスク登録をゲートに統合**する。現行の「振り返りをして明日のタスクを登録してから
  パスワードを生成する」という運用を継続・統合する。MVP はルールの**手動チェック条件**（ラベル付き
  チェックボックス）で代替し、最終形は**振り返りエディタ＋タスクカンバン**をアプリに内蔵して、
  「翌日タスク登録済み＋振り返り済み」を**自動判定**してパスワードのゲートにする。
- **自動行動記録（Google カレンダー1日ビュー風のタイムライン）**を追加する。その日の Edge グループの
  セッションを**グループ色のブロック**で自動生成し、同時進行は同じ時間帯を**分割表示**。ブロックをクリックして
  分配割合（既定 1/N を任意に、0 も可）を編集できる。PC から取得できない時間（昼食・昼寝等）は手動で追記でき、
  空き時間が可視化され、翌日の改善に使える。
- **日次の作業量ダッシュボード**を追加する。当日の内訳を**円グラフ**、日ごとの推移を**棒グラフ**で表示する
  （同時進行は divide-by-N 後の値で、合計は実作業時間と一致）。

## Capabilities

### New Capabilities

- `edge-activity-tracking`: Chromium 拡張機能で「アクティブタブのグループ」を検出し、ローカルアプリへ
  送信。バックエンドがグループ別アクティブ秒数を日次集計。スリープ／アイドル／ロックを除外し、
  複数ウィンドウ・複数グループ環境では最前面のアクティブグループのみを計上（未グループは "その他" バケット）。
- `activity-categorization`: タブグループ→活動カテゴリのマッピング設定、「作業とみなすカテゴリ」の指定、
  グループ別秒数からカテゴリ別秒数および総作業時間へのロールアップ。
- `work-rules-engine`: 日次ルールセット（カテゴリ別・総合のしきい値によるアンロック条件）。
  当日ルールは凍結（変更不可）、翌日以降は編集可能。当日の集計に対する条件評価と達成判定。
- `password-gate`: アンロック条件達成時に、差し替え可能なコマンドで前日・当日のパスワードを生成し表示。
  達成・表示の履歴を記録。平文パスワードの取り扱い方針を含む。
- `activity-timeline`: タブグループのセッションから日次タイムラインを自動生成し、PC外活動を手動追記。
  セッション間のギャップ（空き時間）を可視化し、振り返りに供する。
- `work-stats-dashboard`: 日次のカテゴリ別作業時間を棒グラフで表示するローカルWebダッシュボード。
- `reflection-and-planning`: 日次振り返りの記録と、翌日タスクを登録するカンバン。「翌日計画完了」シグナル
  （振り返り済み＋翌日タスク登録済み）を提供し、パスワードのアンロック条件（PLANNING）に用いる。統合前は手動チェック条件で代替。

## Impact

- **新規コンポーネント**:
  - Chromium/Edge 拡張機能（Manifest V3, TypeScript）: `chrome.tabGroups` / `chrome.tabs` /
    `chrome.windows` / `chrome.idle` / `chrome.alarms` / `chrome.storage` を使用し、
    localhost WebSocket でローカルアプリへイベント送信。
  - ローカルバックエンド（Node.js + TypeScript / Fastify）: WebSocket受信・時間集計・ルール評価・
    パスワード呼び出し・タイムライン生成・統計API。
  - ローカルWebダッシュボード（統計の棒グラフ、タイムライン、ルール編集UI。Chart.js を同梱＝CDN不使用）。
  - データストア（SQLite / better-sqlite3）: セッション/日次集計/ルール/カテゴリ/アンロック履歴/タイムライン。
- **外部依存**:
  - 既存のパスワード生成ロジック（**PowerShell (.ps1)** を差し替え可能なコマンドとして subprocess 統合。中身は本変更では作らない）。
  - ゲームPC側のパスワード変更（既存のタスクスケジューラ。本アプリは原則「表示のみ」で変更はしない）。
- **プラットフォーム**: Windows 11 / PowerShell 環境。Edge（Chromium）。
- **セキュリティ/プライバシー**: シングルユーザー・ローカル完結。ネットワークは localhost のみ。
  パスワードは表示に必要な最小限の扱いにとどめる（永続保存は避ける方針）。
- **依存ライブラリのバージョン**: 実装着手前に公式レジストリで最新安定版を確認して固定する。
  主要: Node.js `>=22` / TypeScript / Fastify / @fastify/websocket / @fastify/static /
  better-sqlite3 / croner / zod（＋ Chart.js 同梱）。確認済みバージョンと固定方針は design.md 参照。

## 確定した設計判断（Decisions）

初期の未確定事項はユーザー確認により以下で確定した（詳細な根拠は design.md）。

1. **構成/パスワードの扱い**: ゲームPCは作業PCと**別マシン**。パスワード変更は各PCのタスクスケジューラが担当し、
   本アプリは条件達成時に**前日・当日の2候補を表示するだけ**（＝表示のみ、アプリはパスワードを変更しない）。
2. **パスワードロジックの形式**: **PowerShell (.ps1)**。「日付を渡すとパスワードを返す差し替え可能なコマンド」として
   subprocess 統合する（内部実装は本 change では作らない）。
3. **技術スタック**: **Node.js + TypeScript**（Fastify）。拡張機能・バックエンドを1言語で統一し、
   ハートビートのペイロード型を zod スキーマとして共有する。データは SQLite（better-sqlite3）。
4. **Edge 検出方式**: **Chromium 拡張機能（MV3）**を作成（外部から確実に「アクティブタブのグループ」を取得できる唯一の現実解）。
5. **脱出弁（break-glass）**: **設けない**。条件未達時のロックアウトのリスクは受容する（＝本アプリの存在意義そのもの）。
   唯一の逃げ道は、ユーザーが意図的に .ps1 を手実行する手間のかかる行為のみ。
6. **1日の区切り**: **午前4時（day_boundary=04:00）**。0:00〜4:00の作業は前日に帰属させる。
7. **実装着手タイミング**: 「Python の課題6」完了後に着手予定（本 change は設計・計画の先行作成）。
