## Why

現状の SPA は 6 タブ（ダッシュボード / ゲート / 当日チェック / タイムライン / 振り返り&カンバン / 設定）で、ゲート条件・当日チェック・PLANNING が重複表示され、行う操作も直感的でない。加えてタイムラインとカンバンの見た目が破綻しており（横伸び・文字切れ・グループ順の入替、浮いたカード）、日々のゲート運用（作業計測→振り返り→翌日タスク登録→パスワード解錠）の体験を損なっている。参照実装 `ref/timeline/`（Google カレンダー風）と `ref/kanban/`（Cadence Board）を基に、情報設計と主要画面を作り直す。

## What Changes

- **情報設計の集約**: ダッシュボード + ゲート + 当日チェック を 1 タブ「今日」に統合。手動チェック（MANUAL_CHECK）は条件リスト内のインラインチェックボックスとして表示し、重複を解消。結果タブ構成は **今日 / タイムライン / カンバン / 振り返り / 設定** の 5 タブ。
- **ダッシュボードの整理**: 「除外内訳」カードを **削除**（ギャップ/離席はタイムラインで可視化済みのため冗長）。総作業時間・グループ別ドーナツ・直近7日棒グラフは「今日」に残す。
- **初回オンボーディング**: 当日ルールが存在せず、かつ未来ルールも未作成の場合、起動時に「初期ルールを作成してください」ダイアログを表示し、翌日ルール作成フローへ誘導する。
- **タイムライン全面刷新**: `ref/timeline/` を移植し、Google カレンダー風の単一縦カラム・時刻ガター・境界目盛り・現在時刻ライン・重なりの列分割レイアウトに再構築。文字切れ・グループ順入替・横伸びを解消。
- **タイムラインの離席記録 UX**: 「離席として記録」ボタンを廃止し、**空きギャップをマウスドラッグして記録**する操作に置換。ドラッグは近傍のブロック端／30分グリッドに吸着し、離すと開始/終了の時刻入力・カテゴリチップ・自由メモ入力のポップオーバーで確定できる。
- **カンバン全面刷新**: `ref/kanban/`（Cadence Board）の見た目・操作を移植。列は **保留 / 未着手 / 進行中 / 完了**、カードは優先度（高/中/低）・期限・Markdown ノートを持ち、ドラッグ&ドロップで移動、完了ドロップで祝福演出＋自動アーカイブ、進捗ドーナツと当日アクティビティログを表示。
- **振り返り刷新**: カンバンとタブ分離。上部にその日の **満足度 5 段階評価**、下部に **Markdown ライブプレビュー** エディタ。保存した振り返りは日付一覧から後から参照可能にする。
- **ゲート整合性の維持**: PLANNING シグナル（振り返り完了＋翌日タスク登録）と MANUAL_CHECK 評価は現状の契約を保つ。カンバンは「翌日期限のタスク数」で PLANNING を満たせるようにする。
- **注記（質問への回答）**: 使用時間はEdge拡張が起動中のみ30秒周期で計測し、サーバー停止中は `chrome.storage.local` に最大2000件（約16時間）退避→再接続時に集計されるため「見たいときだけ `npm run server`」で概ね成立する。ただし04:00の日次ロールオーバー/ルール凍結はサーバー常駐前提。この点を設定画面/README の運用メモに明記する。

## Capabilities

### New Capabilities
- `today-hub`: ダッシュボード・ゲート・当日チェックを 1 タブに集約した「今日」画面。総作業サマリ、条件進捗（手動チェックはインライン操作）、パスワード解錠、除外内訳の削除を含む。
- `onboarding-initial-rules`: 当日ルール未設定かつ未来ルール未作成の初回に、初期ルール作成を促すダイアログと誘導。
- `timeline-calendar`: Google カレンダー風の縦型タイムライン。列分割レイアウト、現在時刻ライン、ギャップのドラッグ記録＋吸着スナップ＋確定ポップオーバー。
- `kanban-board`: Cadence Board 風カンバン。4 列・優先度・期限・Markdown ノート・D&D・完了演出・進捗ドーナツ・アクティビティログ。
- `reflection-journal`: 満足度 5 段階＋Markdown ライブプレビューの振り返りと、過去振り返りの参照。

## Impact

- **フロントエンド（主）**: `server/static/index.html`（タブ再編）、`server/static/js/main.js`（画面登録差し替え）、`server/static/css/app.css`（大幅追加）。新規/刷新 JS: `today.js`（旧 dashboard+gate+checks 統合）、`timeline.js`（全面書き換え）、`kanban.js`（旧 planning から分離・刷新）、`reflection.js`（新規）。`dashboard.js`/`gate.js`/`checks.js`/`planning.js` は統合・置換。
- **バックエンド（副）**: 振り返りに満足度カラム追加と一覧取得エンドポイント（例 `GET /api/reflections`, `PATCH /api/reflection/{date}` の satisfaction 対応）。カンバンのカード拡張（priority / due / notes）に伴うタスクスキーマ拡張とマイグレーション。既存 `/api/tasks`・`/api/planning`・`/api/timeline/*`・`/api/checks/*`・`/api/unlock/*` の契約は維持。
- **参照実装は React + 独自 `support.js`。本アプリは vanilla JS のため、コードを直輸入せず設計・挙動を移植する。**
- **リスク**: (1) PLANNING シグナルとカンバン列/期限の整合、(2) タイムラインのドラッグ操作とスクロール/クリックの競合、(3) タスクスキーマ変更のマイグレーション。詳細は design.md。
