# ユーザーフロー（goal-rule-lifecycle / issue #59）

実際の画面（タブ）を歩いて、目標のループを1周する。
**題材**：目標「髪質を改善する」で、ボリュームアップシャンプーに賭けて、写真と質問で答え合わせする。
途中で壊れたルールを直し、1ヶ月後に「続けるか終えるか」を決める。

タブ構成（現状）：`今日` / `タイムライン` / `カンバン` / `振り返り` / `目標` / `設定`

今回の変更で、#54 の **Plan / Check** は **ルールの追加・変更・削除**に畳まれる。
「賭けを書く→答え合わせを仕掛ける」の2段階が、「ルールを足す（なぜならこう思うから＝理由）」の1段階になる。
ルールを足せる場所は **目標作成時** と **振り返りタブの目標コーナー** の2つだけ（今日タブのルール操作は無くなる）。
ルールの **追加・変更・削除はどれも理由が必須**（理由はルール一覧の各行と⑤沿革に残る）。

❓ は「まだ決まっていない・要判断」の印。

## 目標を作成するとき

*（既存フロー。ステップ3が変わる）*

1. 目標タブ：「＋ 目標を作成」を押す
2. 目標タブ：名前「髪質を改善する」・目的の一文・開始日（今日から／明日から）を入れる
3. 目標タブ：**ルールをその場で作る**（＝この目標で守ること）

   > **変更点**：これまでの「既存ルール条件から選んで**採用**する」画面は無くなる。
   > 代わりに、その目標のためのルールをここで直接足す。足したルールは自動でこの目標に紐づく。
   > 「採用」という言葉・操作は消える。

4. 目標タブ：（任意）初日の写真をステージングする
5. 目標タブ：作成 → 一覧に「髪質を改善する [Day 1/30]」カードが現れる

## ルールを足すとき（賭けを仕掛ける）

*（★#59 の中心。旧「Plan を立てて Check を仕掛ける」がこれになる）*

1. 振り返りタブ：いつもどおり今日の振り返りを書く
2. 振り返りタブ：目標「髪質を改善する」のコーナーに **ルール一覧**と **「＋ 追加」** ボタン。押す
3. 振り返りタブ：**1つのフォーム**でルールを1件作る。フォームは **独立した2軸**でできている：

   ```
   ┌──────────────────────────────────────────────────┐
   │ 種類  ⏱総作業  ⏱グループ  ⏱カテゴリ  ☑チェック       │ ← 軸1
   │       ● 📷 写真を出す      ○ 💬 質問に答える          │
   │        撮るもの「前髪・正面」                         │
   │        （質問なら質問文を先に書く）                    │
   ├──────────────────────────────────────────────────┤
   │ いつ  ● ずっと（永続）  ○ 単発  ○ 範囲                │ ← 軸2（種類とは無関係）
   │        単発 … 3日後（or 7/18 と直接指定）             │
   │        範囲 … 今日から 7日間（毎日）                   │
   ├──────────────────────────────────────────────────┤
   │ 理由（必須）                                        │
   │   「ボリュームアップシャンプーを使えば髪質が良くなるのでは」│
   └──────────────────────────────────────────────────┘
   ```

   > **変更点**：旧フォームは「Plan を書く」→「Check を足す」の2段階だった。
   > 今回は Plan 欄が **理由欄**になり、Check の2軸（種類×いつ）がそのままルールの2軸になる。
   > 賭けの一文は理由に溶ける。「単発／範囲」に **ずっと（永続）** が加わる。

   この例では2つ足す：
   - **ルール①**：📷 写真「前髪・正面」／ **範囲**・今日から7日間（毎日）／理由「…髪質が良くなるのでは」
   - **ルール②**：💬 質問「使用感はどうだった？」／ **単発**・7日後／理由「1週間後に手応えを確かめたい」

4. 振り返りタブ：その下の **日記**（既存の自由記入エディタ）は今までどおり書く
5. 振り返りタブ：ルール一覧に足したルールが並び、**最近の変更**にも1行ずつ積まれる

   ```
   ルール
    ⏱ 総作業時間 4時間
    📷 前髪・正面        今日〜+6日（範囲・毎日）      ✎  −
    💬 使用感はどうだった？   +7日（単発）              ✎  −
   最近の変更
    ＋追加 📷 前髪・正面 7日間 ― …髪質が良くなるのではないだろうか
    ＋追加 💬 使用感はどうだった？ 単発
   ```

> **追加・変更・削除はどれも理由が必須**（場合分けしない）。理由はルール一覧の各行と⑤沿革に残る。

## ルールの日が来て、気づくとき

*（範囲ルールの初日・単発ルールの当日。#54 のトースト動線をそのまま引き継ぐ）*

1. **その日はじめてダッシュボードを開いた瞬間、トーストが1回出る**
   > 📷 前髪・正面 ― 今日のルールです（髪質を改善する）
2. 今日タブ：開く。作業時間・内訳を見る
3. 今日タブ：**ゲートの不足条件にルールが並ぶ**
   ```
   総作業時間 4h                        ✅
   翌日の計画                            ✅
   📷 前髪・正面                         ❌ 写真がまだ
   💬 使用感はどうだった？               ❌ 未回答
   ```
4. 今日タブ：他の条件を全部満たしても、**パスワードは表示されない**
5. 今日タブ：不足条件の行から直接 **答えられる**

> 通知の方針は #54 から不変（時刻でスケジュールしない・その日初回オープンで `toast()` 1回・day_key フラグ）。

## ルールに答えるとき

*（#54 の Check 回答と同じ。今日タブから直接）*

### 📷 写真のルール

1. 今日タブ：不足条件の「📷 前髪・正面」から **「写真を出す」**
2. 今日タブ：画像を貼る／ファイルを選ぶ（キャプションは先に決めてあるので確認だけ）
3. 今日タブ：提出 → 条件が ✅ に変わる。範囲なら翌日また出す（③Before/After 列に時系列で積む）

### 💬 質問のルール

1. 今日タブ：不足条件の「💬 使用感はどうだった？」から **「答える」**
2. 今日タブ：質問文が出ているので答えを書く「泡立ちは良い。乾燥は減った気がする」
3. 今日タブ：保存 → 条件が ✅ に変わる。答えは⑤沿革に **Q&A のペア**で載る

## ルールをサボった日／範囲ルールの扱い

*（#54 と同じ。繰り越しは種類で決まる）*

```
📌 単発（📷/💬 のピンポイント1日）
    出すまでロックを繰り越す。7/18 に出さなければ 7/19 も開かない。

📅 範囲（毎日 n 日間）
    その日限りのゲート。サボった日は繰り越さず、翌日は翌日の分だけを要求する。
    範囲を過ぎたら消える。沿革には「7日中5日提出」と事実どおり残る。

⏱ 時間型（総作業・グループ・カテゴリ）
    繰り越しは無い（その日の実測で判定）。
```

## ルールをゆるめる・消すとき（理由つき）

*（旧「Plan / Check をやめる」がこれになる。唯一の脱出弁。逃げた事実が歴史に残る）*

1. 振り返りタブ：目標コーナーのルール一覧で、ゆるめたい・消したいルールの **✎** か **−** を押す
2. 振り返りタブ：**理由の入力を求められる（必須・空は拒否）**
   - 閾値を下げる（例：総作業 4h → 3h）… ✎ ＋理由「課題週間。ゼロにはしない」
   - まるごと消す（例：ブログ 30分）… − ＋理由「反応が薄いから」
3. 振り返りタブ：実行 → **その日から効く**（過去日の判定は書き換えない）
4. 振り返りタブ：**最近の変更／⑤沿革にそのまま残る**（消さない）
   ```
   最近の変更
    ✎変更 総作業時間 4h → 3h ― 課題週間。ゼロにはしない
    −削除 ブログ 30分 ― 反応が薄いから
   ```

> **変更点**：これまで採用中の条件は**削除できなかった**（ジャンル固定）。今回はいつでも変更・削除でき、
> **その日から効く**。理由は追加・変更・削除のどれでも必須。過去に積んだ達成日数は動かない（凍結済み）。

3. 振り返りタブ：**✎** を押す。グループ選択に直近使ったグループ候補が出る
   （**`競技プログラミング` と `面接` は別々に並ぶ**＝group-rule-snapshot-identity 修正済みのため）
4. 振り返りタブ：グループを **`競技プログラミング`** に選び直し、理由「拡張のバグでUUIDが壊れていた」＋**保存**

   - **⑤沿革**：`✎差替 グループ ????? → 競技プログラミング ― 拡張のバグでUUIDが壊れていた`

> **変更点**：これまでは「変更することもできない」状態だった（issue 後半）。今回、同じルールのまま中身を直せる。

## 30日たって、続けるか終えるか決めるとき

*（★新しい動線。完走がループの分岐点になる）*

1. 目標タブ：完走した「英語を毎日やる [完走]」を開くと、レポートの先頭に問いが出る
   ```
   ┌─────────────────────────────────────────┐
   │  英語を毎日やる の1ヶ月が終わりました。    │
   │  続けますか？                            │
   │     [ 続ける ]        [ ここで終える ]    │
   └─────────────────────────────────────────┘
   ```
2a. **続ける** を押す：
   - 新しい30日目標が **作り直される**（Day 1/30 にリセット・名前と目的は引き継ぎ確認）
   - 永続ルール「⏱ 英語 60分」は **そのまま続投**（ゲートに残り、新目標に紐づく）
   - 前サイクルの完走レポートは読めるまま残る
2b. **ここで終える** を押す（理由は任意）：
   - 永続ルールが **ゲートから外れる**（もう縛られない）
   - 目標は「完走・終了」として **アーカイブ**。レポート・沿革・カレンダーは **読めるまま残る**
   - 理由を書けば、沿革の最後に `英語を毎日やる をここで終える ― （理由）` が残る
3. どちらも押さず放置：**永続ルールはゲートに残り続けて縛る**（惰性フェードアウトはできない）

> **変更点**：これまで完走後は「レポートを開く」だけだった。今回、完走が「続ける／終える」の分岐点になる。

*（★新しい動線。目標の終わりを越えるルールを作ったとき）*

1. 振り返りタブ：目標「髪質を改善する（7/24 完走予定）」で、7/22 に 📷 範囲ルールを「7日間」（＝7/28 まで）で足そうとする
2. 保存時、システムが「このルールは目標の終了を越える」と気づいて問う：
   ```
   ┌──────────────────────────────────────────┐
   │  この写真ルールは 7/28 まで続きます。       │
   │  「髪質を改善する」は 7/24 で終了予定です。  │
   │  目標の終了を 7/28 まで伸ばしますか？        │
   │     [ 伸ばす ]        [ やめておく ]        │

   ```
3a. **伸ばす**：目標の終了が **7/28 に延びる**。表示は **Day N/M**（30/30 → 34/34）。
    カレンダーの列が増え、完走（と続ける/終える の問い）が 7/28 の後にずれる
3b. **やめておく**：ルールを **目標末尾（7/24）まで自動で切り詰める**（7日間 → 3日間）。
    「目標末尾まで（3日間）に調整しました」と注記して、作成自体は通す

> **変更点**：これまで期間は「30日固定」だった。今回、末尾のルールを見届けるために **前方向にだけ**伸ばせる
> （短縮の手段は無い・走行中の目標放棄には使えない）。

## 今回やらないこと

- ルールの共有（同一内容ルールの dedup）。複数目標が同内容ルールを足すと2行になるが、判定は壊れない
- `daily_totals_snapshot` の生データ・総作業時間の算入スコープ・divide-by-N 配分・日境界分割の変更
- group identity レジストリ・改名追随・拡張の採番修正（`group-rule-snapshot-identity` で完了済み。本変更はその上に乗る）

## 決まったこと

- **Plan / Check** の語彙・データモデルは畳む。すべて **ルールの追加・変更・削除** に一本化する
- ルールは **第一級**。中身（閾値・グループ・ラベル等）を変えても **同じルール**として扱い、過去の記録と繋がる
- ルールは **いつでも追加・変更・削除でき、当日から効く**（採用中条件の削除禁止＝ジャンル固定は撤廃）
- ルールを足せる入口は **目標作成時** と **振り返りの目標コーナー** の2つだけ。
  **今日タブのルール操作（作成・編集・削除）は全撤去**し、目標コーナーに一本化する
- 「**採用**」概念は廃止。足した場所の目標に自動で紐づく
- 理由は **追加・変更・削除のどれでも必須**（場合分けしない）。理由は最近の変更／⑤沿革に残る
- ⑤沿革は **ルール操作の年表**（写真＝画像・質問＝Q&A がぶら下がる）。日記は載せない（既存どおり）
- 完走で **「続ける／終える」** を問う。終えても **レポート・沿革は残す**
- 目標末尾を越えるルールを足すと **「目標を伸ばすか」** を問う。伸ばす＝Day N/M・やめる＝末尾まで切り詰め
- **デモモードで日付を進めて**（「1日後」「30日後」ボタン）、単発ルールの通知と完走フォークを体験できるようにする
