## ADDED Requirements

### Requirement: タイムラインタブの振り返りタブへの統合とナビゲーション

システムは独立した「タイムライン」タブを廃止し、その完全版タイムラインを **振り返りタブ**（タブ名は既存どおり「振り返り」）の中のビューとして提供しなければならない（MUST）。統合後のグローバルナビゲーションは `今日 / 振り返り / カンバン / 目標 / 設定` の5タブでなければならない（MUST）。

- タイムラインの記録・編集（離席記録・手動エントリ・削除など既存動線）は、振り返りタブ内の完全版タイムラインビューで従来どおり行えなければならない（MUST）。振り返り内へ簡略化した読み取り専用の複製を別に持ってはならない（MUST NOT）。
- 従来 `#timeline` で始まるディープリンク（通知からの遷移）は、振り返りタブを開き対象のタイムラインビューを表示する経路へ移行しなければならない（MUST。リンク切れにしてはならない＝MUST NOT）。

#### Scenario: ナビゲーションからタイムラインタブが消える
- **WHEN** ユーザーがグローバルナビゲーションを見る
- **THEN** タブは `今日 / 振り返り / カンバン / 目標 / 設定` の5つで、「タイムライン」単独タブは存在しない

#### Scenario: タイムラインの記録は振り返りタブ内で行える
- **WHEN** ユーザーが振り返りタブでタイムラインビューを開き、未記録区間の「＋ 未記録」から離席を記録する
- **THEN** 記録は従来のタイムラインタブと同じ経路で保存され、ビューへ反映される

#### Scenario: 通知ディープリンクが振り返りへ着地する
- **WHEN** `#timeline` で始まる URL でアプリを開く（または遷移する）
- **THEN** 振り返りタブが開き、タイムラインビューが表示される

### Requirement: 左右2分割ワークスペース

振り返りタブは、**左＝メイン領域（大）** と **右＝振り返りサイドバー（常時表示・相対的に細い）** の2分割レイアウトでなければならない（MUST）。

- 左メインでタイムライン等を見ながら、右サイドバーで振り返り本文を書けなければならない（MUST。両者は同時に画面上に見えていなければならず＝MUST、片方をめくると他方が隠れる構造にしてはならない＝MUST NOT）。
- 右サイドバーは左メインでどのビューを表示していても常時表示されていなければならない（MUST）。
- レイアウトは詰め込み（過度に狭いカラムへ要素を圧縮する構成）を避け、各領域は自領域内でスクロールしなければならない（MUST。ページ全体を縦に流す構成にしない）。

#### Scenario: タイムラインを見ながら書ける
- **WHEN** 左メインでタイムラインビューを表示している
- **THEN** 右サイドバーの振り返り本文エディタは同時に見えており、タイムラインを見ながら本文を入力できる

#### Scenario: サイドバーは常時見える
- **WHEN** 左メインを「一日の配分」や「明日の計画」へ切り替える
- **THEN** 右の振り返りサイドバーは隠れず、そのまま表示され続ける

### Requirement: 左メインのビュー切替（横スクロール）

左メインは `タイムライン（完全版） / 一日の配分 / 明日の計画` の3ビューを **横スクロール（スナップ）でめくる** ビュー切替として提供しなければならない（MUST）。

- ビューはラベル付きの切替コントロール（タブ）と横スクロールの双方で移動できなければならない（MUST）。現在ビューは視覚的に明示されなければならない（MUST）。
- 各ビューは全幅で余白を持って表示し、狭い枠へ複数ビューを同時に圧縮してはならない（MUST NOT）。
- ビュー切替は対象日を変えてはならない（MUST NOT。対象日は日付ストリップが駆動する）。

#### Scenario: 3ビューをめくる
- **WHEN** ユーザーが切替コントロールで「一日の配分」を選ぶ、または左メインを横スクロールする
- **THEN** 左メインは対応するビューへスナップして切り替わり、現在ビューが明示される

#### Scenario: ビュー切替は対象日を保つ
- **WHEN** 対象日が 7/24 のまま左メインを別ビューへ切り替える
- **THEN** 対象日は 7/24 のままで、切替先ビューも 7/24 のデータを表示する

### Requirement: 日付ストリップによる対象日ナビゲーション

対象日ナビゲーションは、前後の日を横に並べた **日付ストリップ** として画面上部に提供しなければならない（MUST）。ストリップは画面全体（左メインの全ビュー＋右サイドバー）の対象日を駆動しなければならない（MUST）。

- 各日チップは曜日・日番号を表示し、その日の振り返りの **気分を色ドット** で示さなければならない（MUST。未記入の日はドット無し等で区別する）。未来日（＝明日の計画の対象になり得る日）は現在日・過去日と視覚的に区別しなければならない（MUST）。
- 前後移動の矢印（‹ ›）、チップの直接選択、「今日」への即時復帰を提供しなければならない（MUST）。
- 対象日を変えると、左メインの現在ビューと右サイドバーの本文・気分・目標日記が対象日のデータへ同期しなければならない（MUST）。
- このストリップは、従来 `reflection-journal` の右レールにあった「過去の振り返り一覧（日付選択）」の役割を引き継ぐ（別途、日付ピッカーからの任意日ジャンプを併設してよい）。

#### Scenario: 気分ドット付きで前後日が見える
- **WHEN** ユーザーが振り返りタブを開く
- **THEN** 上部に前後数日の日付チップが並び、各日の振り返りの気分が色ドットで示され、未記入日はドット無しで区別される

#### Scenario: チップ選択で全体が同期する
- **WHEN** ユーザーが別の日付チップを選ぶ
- **THEN** 左メインの現在ビュー（タイムライン/配分/明日の計画）と右サイドバーの本文・気分・目標日記が、その日のデータへ更新される

#### Scenario: 「今日」で復帰する
- **WHEN** 過去日を表示中に「今日」を押す
- **THEN** 対象日が当日に戻り、全ビューが当日のデータへ更新される

### Requirement: ワークスペースの視覚的忠実性

統合ワークスペースの見た目は参照実装 `ref/reflection/`（統合後の HTML）に忠実でなければならない（MUST）。配色 hex・寸法・要素アナトミーは参照の値を `rf-*` CSS クラスへ転記する。CSP（`style-src 'self'`）のためインライン `style=` 属性を使ってはならず（MUST NOT）、静的スタイルはクラス、動的値のみ CSSOM で適用する（**フォントのみ** `system-ui` フォールバックを許容する）。受け入れ基準の詳細な画面遷移は companion `ユーザーフロー.md` を唯一の正典とする。

#### Scenario: 参照実装との一致
- **WHEN** 統合ワークスペースを表示する
- **THEN** 左メイン（ビュー切替・横スクロール）＋右サイドバー＋日付ストリップの配色・寸法・レイアウトが `ref/reflection/` の参照実装どおりに描画される

#### Scenario: CSP 適合
- **WHEN** ワークスペースの DOM を検査する
- **THEN** インライン `style=` 属性は存在せず、スタイルは `rf-*` クラスと CSSOM のみで適用されている
