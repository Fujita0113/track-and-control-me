## MODIFIED Requirements

### Requirement: 設計への視覚的忠実性

振り返りタブ（統合ワークスペース）の見た目は参照実装 `ref/reflection/`（統合後の HTML）に忠実でなければならない（MUST）。配色 hex・寸法・要素アナトミーは参照の値をそのまま `rf-*` CSS クラスへ転記する。CSP（`style-src 'self'`）のためインライン `style=` 属性を使ってはならず（MUST NOT）、静的スタイルはクラス、動的値のみ CSSOM で適用する。**フォントのみ** CSP 制約により `system-ui` フォールバックを許容する。

本文エディタと5段階「気分」入力は、**右の振り返りサイドバー**内に配置しなければならない（MUST）。従来の「左本文＋右レール（300px）の2カラム」構成は、`reflection-timeline-workspace` の「左メイン（大）＋右サイドバー」構成へ置き換わる。詳細な画面遷移・受け入れ基準は companion `ユーザーフロー.md` を唯一の正典とする。

#### Scenario: 配色・レイアウトの一致
- **WHEN** 振り返りタブを表示する
- **THEN** 右サイドバー内に本文エディタカード（白・薄いボーダー・角丸）と気分入力・下部クローム（文字数・保存）が参照どおりに描画され、左メインのビュー切替・日付ストリップと調和した配色で表示される

#### Scenario: CSP 適合
- **WHEN** 振り返りタブの DOM を検査する
- **THEN** インライン `style=` 属性は存在せず、スタイルは `rf-*` クラスと CSSOM（`element.style.*`）のみで適用されている

## REMOVED Requirements

### Requirement: 過去の振り返り一覧（抜粋付き）

**Reason**: 対象日ナビゲーションを `reflection-timeline-workspace` の **日付ストリップ**（気分ドット付き）へ一本化するため。右レールの一覧UIは統合ワークスペースでは廃止する。

**Migration**: 過去日の閲覧・選択は日付ストリップの日付チップ（前後移動・直接選択・「今日」復帰、必要に応じ日付ピッカー併設）で行う。`GET /api/reflections`（`excerpt` を含む）は日付ストリップの気分ドット等のために引き続き利用してよい（API 自体は変更しない）。
