# ユーザーフロー（reflection-timeline-workspace / issue #61）

実際の画面（タブ）を歩いて、一日の締め（振り返り→明日の計画）を1周する。
**題材**：7/24（木）。今日は午前に大学、午後に #61 モック、夕方に面接対策。
明日の段取りは「まず9時までに大学、次に筋トレ、タスク確認してから、課題があれば課題／なければ開発＋OpenWork 面接対策」。

タブ構成（**変更後**）：`今日` / **`振り返り`** / `カンバン` / `目標` / `設定`
→ ★**「タイムライン」単独タブは廃止**し、完全版タイムラインは振り返りタブの中のビューになる。タブ名は既存どおり「振り返り」。

## 骨格

「今日の作業記録（タイムライン）を**見ながら**振り返りを書き、書いた明日の段取りを**見ながら**カンバンへ登録する」を、そのまま1画面に出す。

```
┌─ 振り返りタブ ─────────────────────────────────────┐
│ [日付ストリップ] ‹ 19 20 21 22 23 [24] 25 26 27 › 今日 │
├──────────────────────────────────┬────────────────┤
│ 左＝メイン（大）横スクロールでめくる │ 右＝振り返り（小・常時）│
│  ▤ タイムライン / ◑ 一日の配分 / ✎ 明日の計画        │ 気分 ●●●●●     │
│                                    │ 本文エディタ    │
│  ← 完全版を “どん” と全幅で ―― →     │ （左を見ながら書く）│
│                                    │ 目標の日記      │
└──────────────────────────────────┴────────────────┘
```

- **左（大）と右（小）は常に同時に見える。** 片方をめくると他方が隠れる構造にはしない（＝「見ながら」が壊れない）。
- 左メインだけが `タイムライン / 一日の配分 / 明日の計画` を横スクロール（スナップ）でめくる。
- 対象日は**日付ストリップが単独で駆動**する。ビューをめくっても対象日は変わらない。

画面の分業（変更後）：

```
左メイン … 見る／めくる（完全版タイムライン・一日の配分・明日の計画）
右サイドバー … 書く（気分・本文・目標日記。今までどおり、置き場所だけ右へ）
日付ストリップ … 選ぶ（対象日を全体へ流す。過去の閲覧もここ）
```

❓ は「まだ決まっていない・要判断」の印。

## 振り返りタブを開くとき

1. ナビの「振り返り」を押す（または通知の `#timeline` ディープリンクで着地）
   > ★**変更点**：`#timeline` はこれまで独立タイムラインタブへ飛んでいた。今後は**振り返りタブを開き、左メインをタイムラインビューにして**着地する（リンク切れにしない）。
2. 左メインは既定で **① タイムライン（完全版）**、右サイドバーに本文エディタ、上部に日付ストリップ（既定＝今日 7/24）
3. 日付ストリップの各日チップには**気分の色ドット**が付く（緑=良い等・未記入日はドット無し・未来日は破線で区別）
   > ★**変更点**：従来の右レール「過去の振り返り一覧（抜粋付き）」は**廃止**。過去日の閲覧・選択は日付ストリップへ役割移管（`GET /api/reflections` はドットの気分表示に流用・API 自体は不変）。

1. 左メイン ① タイムライン：今日の作業記録が**完全版**で見える
   ```
   9  ┃ 大学・線形代数の課題   09:05–10:40
      ┃ ＋未記録 25分
   11 ┃ ゼミ打合せ
      ┃ 昼休憩

   ```
   > ★**変更点**：これは簡略リボンではなく**完全版タイムラインそのもの**。離席記録（＋未記録のドラッグ）・ポップオーバー等、従来のタイムラインタブでできた操作はここで**そのまま**できる（読み取り専用の複製ではない）。
2. 右サイドバー：左を見ながら、気分ピルを選び、本文を書く
   > *本文エディタ（`md-editor`）・気分・保存動線は**今までどおり**。置き場所が右サイドバーになっただけ。*
   ```
   今日できたこと … 午前は大学、午後は #61 モック。夕方に失速→仕上げ。
   面接対策は開発と並行で1時間。
   明日の段取り：まず9時までに大学、次に筋トレ。タスク確認してから、
   課題があれば課題／なければ開発＋OpenWork 面接対策。
   ```
3. 右サイドバー：目標の日記コーナー（進行中の目標ぶん）も今までどおり下に続く
4. 保存（手動ボタン、または日付切替・タブ離脱でフラッシュ）
   > *`content.trim().length>0` を「振り返り完了」とみなす PLANNING シグナルの意味は不変。*

## 一日の配分を見るとき（ビューをめくる）

1. 左メインの切替タブ「◑ 一日の配分」を押す（または左メインを横スクロール）
2. 左メインが**独立した全幅ビュー**として配分バーを表示
   ```
   開発     ▬▬▬▬▬▬▬▬▬▬  3h40m
   大学     ▬▬▬▬▬▬       2h10m
   面接対策 ▬▬▬▬          1h15m
   休憩     ▬▬▬           1h05m
   未記録   ▬▬            55m   ← 常に最下部・中立色
   ```
   > ★**変更点**：配分はこれまで**本文エディタ上部の常設バー**だった。これを左メインの**独立ビュー**へ昇格（エディタ上部からは消える）。集計規則（持ち分・名前＋色 identity・端〜端の母数・降順＋未記録最下部）は**不変**。
3. 右サイドバーの本文は**見えたまま**（めくっても隠れない）
4. 切替で対象日は変わらない（7/24 のまま配分を見ている）

## 明日の計画を書いて、登録するとき（★見ながら一個ずつ）

*（issue #61 の核その2。往復をなくす）*

1. 左メインの切替タブ「✎ 明日の計画」を押す
   > ★**変更点**：従来の「振り返りを終えて明日の計画へ →」ボタンで**カンバンタブへ画面遷移して手入力**していた動線は**廃止**。振り返りタブ内で完結する。
2. 右サイドバーの本文（さっき書いた「明日の段取り」）は**右に見えたまま**。左でそれを見ながら登録する
3. 左メイン：本文の明日段落から拾った**候補チップ**が並ぶ。タップで即登録（タイプ不要・書いた言葉がそのままタスク名）
   ```
   本文から拾えるもど（タップで追加）：
     [＋ 大学に行く] [＋ 筋トレ] [＋ タスク確認] [＋ 課題を確認]
   ```
   > 候補抽出は素朴なヒューリスティック（句読点・接続詞で分割した短句）。誤抽出しても**タップしなければ入らない**ので安全。
4. 左メイン：大きな入力欄に直接打って **Enter で連続追加**もできる
   > IME 変換確定の Enter では登録しない（`enter-submit-ime-guard` と整合）。
5. 登録済みは**余白のある縦リスト**で下に積まれる（詰め込み列ではない）
   ```
   ⠿ ● 9時までに大学に行く            [明日]  ×
   ⠿ ● 筋トレ                          [明日]  ×
   ⠿ ● タスク確認                      [明日]  ×
   ⠿ ● 課題あり→課題／なし→開発＋面接対策 [分岐]  ×
   ```
   - ドラッグで並べ替え（カンバン未着手列の within-column 順序と一致）
   - × で取り消し（カンバンからも消える）
6. ここで登録したものは**本物のカンバン**の未着手（TODO）列・`due = 7/25`（翌日）の未完了タスクになる
   > ★別データストアは作らない。カンバンタブの「明日のタスク n/N（`due=翌日 かつ 未完了`）」進捗と**同じ集合**。カンバンタブを開けば同じタスクがそこにある。

> **分岐（課題があれば…）の扱い**：条件付きは **1タスクのテキストとして残す**だけ（例「課題あり→課題／なし→開発＋面接対策」）。ルール化・実行時評価は**しない**（朝、自分で読んで判断する）。❓ この割り切りでよい？

## 過去の日を読み返すとき（日付ストリップ）

1. 日付ストリップで別日チップ（例 7/22）を選ぶ、または ‹ › で1日ずつ
2. **左メインの現在ビュー**（タイムライン／配分／明日の計画）と**右サイドバー**（本文・気分・目標日記）が、その日のデータへ一斉に同期
3. 「今日」ボタンで当日（7/24）へ即復帰
   > ★**変更点**：従来は右レールの過去一覧クリック＋素の `<input type=date>` で日付を変えていた。これを気分ドット付きストリップへ一本化（任意日ジャンプ用の日付ピッカーは併設してよい）。

## 今回やらないこと

- タイムラインのラン結合・離席・同時記録などの**描画規則そのもの**の変更（`timeline-run-view` 他は不変。ホストする場所が変わるだけ）
- 配分集計・`reflection_entry` の意味・カンバンの列/並べ替え/詳細エディタの**仕様変更**
- 明日タスクの**分岐の実行時評価**（1タスクのテキストに留める）
- 候補チップの**形態素解析・LLM 整形**（素朴なヒューリスティックのみ・完全ローカル原則を維持）
- 新規サーバ API の追加（既存 read/task API の再利用で足りる）

## 決まったこと

- タイムラインタブを**振り返りタブへ統合**。ナビは5タブ（タブ名は「振り返り」のまま）
- レイアウトは **左＝タイムライン（大）／右＝振り返り（小・常時）**。同時に見えて「見ながら書く」が成立
- 左メインは **横スクロールでビュー切替**：タイムライン（完全版）／一日の配分／明日の計画
- **一日の配分はエディタから分離**して独立ビューへ。**右オーバーレイの簡略リボンは廃止**
- 対象日は**気分ドット付き日付ストリップ**が単独駆動。**過去の振り返り一覧は廃止**しストリップへ統合
- 明日の計画は**振り返りタブ内で完結**。候補チップのタップ or 直接入力で**本物のカンバンへ `due=翌日`**。詰め込み列は使わない
- 分岐は**1タスクのテキスト**で残す（ルール化しない）
- 受け入れ基準はこの `ユーザーフロー.md` を唯一の正典とし、視覚要件は `ref/reflection/` 統合参照へ写経＋スクショ検証
