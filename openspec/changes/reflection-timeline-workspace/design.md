## Context

現状、振り返り（`reflection.js`）とタイムライン（`timeline.js`）は別タブで、振り返り内のタイムラインは右オーバーレイの簡略リボン（`buildRibbon`）に留まる。配分バー（`renderAlloc`）は本文エディタ上部に常設。明日タスク登録は `goToPlanning()` が `setTomorrowMode(true)` の後にカンバンタブへ画面遷移し、そこで手入力する。結果として「タイムラインを見ながら書く」「書いた段取りを見ながら登録する」がどちらも成立していない（issue #61）。

フロントは CSP 制約下のバニラ JS（`server/static/js/*`、モジュールは `show(section)/hide()` 契約）。各画面は `main.js` の `SCREENS` でルーティングされる。視覚要件は `ref/` への写経＋スクショ検証で担保する運用（別物事故の再発防止）。受け入れ基準の画面遷移は companion `ユーザーフロー.md` を唯一の正典とする。

探索の到達点は公開モック（案：左タイムライン大／右振り返りサイドバー、左メインは Notion 風ビュー切替の横スクロール、明日の計画は「拾って積む」クイック登録）。本設計はそれを実装アーキテクチャへ落とす。

## Goals / Non-Goals

**Goals:**
- タイムラインタブを振り返りタブへ統合し、ナビゲーションを5タブ化する。`#timeline` ディープリンクを振り返りへ移行。
- 左メイン（大）＝ビュー切替（タイムライン完全版／一日の配分／明日の計画）の横スクロール・デッキ、右サイドバー（細・常時）＝振り返り本文＋気分＋目標日記。
- 対象日を日付ストリップ（気分ドット付き）で駆動し、全ビュー＋サイドバーへ同期。
- 明日の計画を振り返りタブ内で完結（本物のカンバンへ `due=翌日` で登録）。
- 既存の集計・データ・エディタ挙動・カンバン仕様を不変に保つ（**表示レイヤーの再配置**に徹する）。

**Non-Goals:**
- タイムラインのラン結合・離席・同時記録などの描画規則（`timeline-run-view` 他）の変更。
- 配分集計・`reflection_entry` の意味・カンバンの列/並べ替え/詳細エディタの仕様変更。
- 新規サーバAPIの追加（原則、既存 read/task API の再利用で足りる）。
- 明日タスクの「分岐（条件付き）」の実行時評価。分岐は1タスクのテキストとして残すに留める（ルール化はしない）。

## Decisions

### D1: `reflection.js` を器化し、`timeline.js` の描画を**関数として埋め込む**（iframe やタブ複製をしない）
`reflection.js` の `show()` を「左メイン（ビュー切替コンテナ）＋右サイドバー」を組む器に作り替える。タイムラインビューは `timeline.js` の描画本体を **`render(container, date)` として呼べる形に抽出**して左メインへマウントする。
- 代替案: タイムラインを iframe や別 section の使い回しで載せる → DOM/イベントの二重管理・CSP・スクロール制御が複雑化するため却下。
- 代替案: 簡略リボンを流用 → issue の「完全版を見たい」に反するため却下（`buildRibbon`/`renderTimelinePanel` は撤去）。
- 影響: `timeline.js` は現在 `show(root)` 契約。記録・編集・離席ドラッグ等の副作用を保ったまま「対象日を引数に取り任意コンテナへ描画する」内部関数へリファクタする。`main.js` の `SCREENS` からは `timeline` を除去。

### D2: 対象日は**ワークスペースが所有する単一の `activeDate`**。ビューは対象日に対する純粋な描画者
日付ストリップが `activeDate` を更新 → 現在ビューの `render(container, activeDate)` と右サイドバーの本文ロードを再実行。ビュー切替は `activeDate` を変えない。これにより「配分だけ別日」等の不整合を排除。
- 気分ドットは `GET /api/reflections`（`excerpt`＋`satisfaction`）から前後日ぶんを引く。API は不変。
- 代替案: 各ビューが自前の日付状態を持つ → 同期バグの温床。単一所有に統一。

### D3: 左メインは **CSS scroll-snap の横スクロール**、ビュータブは `scrollIntoView` のジャンプ
`overflow-x:auto; scroll-snap-type:x mandatory`＋各ビュー `scroll-snap-align`。タブ/矢印はスクロール、現在ビューはスクロール位置から逆算してハイライト（モック実装済みのパターン）。各ビューは内部縦スクロール。ページ body は縦に流さない（枠内スクロール）。
- タイムラインは重い描画になり得るため、**現在表示中ビューのみ描画/更新**し、非表示ビューは遅延させてよい（初期は全ビュー生成でも可、パフォーマンス問題が出たら遅延化）。

### D4: 明日の計画は**本物のカンバンへ `createTask` するだけ**（別ストア無し）
`kanban.js` の `computeDue(null, 'TODO', /*tomorrowOn*/true, workday)` 相当で `due=翌日` を算出し、`createTask({title, status:'TODO', due, ...})`。カンバンタブの「明日のタスク n/N（`due=翌日 かつ 未完了`）」と同集合。並べ替えは `kanban-task-reorder` の within-column 順序を使う。
- 候補チップ抽出は**素朴なヒューリスティック**（本文の明日段落を句読点・接続詞で分割し短句を候補化）。形態素解析等は導入しない。誤抽出は「タップしなければ入らない」ため安全側。
- `goToPlanning()`＋カンバン遷移動線は撤去し、振り返りタブ内で完結。`setTomorrowMode` は明日の計画ビュー滞在中の既定として内部利用してよい。
- 代替案: 明日タスク専用テーブル → カンバンと二重管理になり却下（issue は「本物のカンバンに登録」）。

### D5: 右サイドバーは常時表示の固定幅。本文エディタ（`md-editor.js`）・気分・目標日記（`journalCorner`）を移設
`createMarkdownEditor` と保存動線（`doSave`/`flush`/`putReflection`/`putGoalJournal`）はそのまま。配置のみサイドバーへ。右レール「過去の振り返り一覧」は撤去し日付ストリップへ役割移管。
- 明日の計画ビュー滞在時、本文の「明日の段取り」段落を参照しやすく強調してよい（任意）。

### D6: `ref/` を統合ワークスペースへ更新し、スクショ検証
`ref/reflection/` に統合後 HTML を写経（左メイン＋右サイドバー＋日付ストリップ）。既存 `ref/timeline/`・`ref/reflection/振り返り.dc.html` の視覚要件は統合参照へ束ねる。実装後にスクショで一致確認（[[reference-impl-in-ref-dir]] の流儀）。

## Risks / Trade-offs

- **[横スクロール内に縦スクロールの2軸操作]** マウスで横移動しづらい → タブ＋矢印＋ドット＋スナップを併設し、横移動を明示導線で担保（ホイール横化には依存しない）。
- **[`timeline.js` のリファクタ副作用]** 離席ドラッグ・ポップオーバー・自動スクロール等の副作用がコンテナ変更で壊れる恐れ → `show(root)` を薄いラッパにして内部 `render(container,date)` へ委譲し、既存 e2e（`e2e/` タイムライン系）で回帰確認。
- **[デモモードの追従漏れ]** デモは閲覧専用で別描画（`showDemo`）→ 新レイアウトへ追従が必要。`demo-seed` の固定 day_key で配分・タイムライン・（閲覧時の）明日の計画非表示を確認（プロジェクト CLAUDE.md のデモ検証ルール）。
- **[候補チップの誤抽出]** ヒューリスティックの過不足 → タップ登録のみ・直接入力併設・登録後編集/削除可、で実害を最小化。
- **[`#timeline` ディープリンク移行漏れ]** 通知からの遷移が切れる → `main.js` の hashchange/initialScreen を振り返り＋タイムラインビュー選択へ差し替え、通知（`plan-check`/hooks）経路を確認。

## Migration Plan

1. `timeline.js` を `render(container, date)` 抽出＋薄い `show()` ラッパにリファクタ（挙動不変を e2e で担保）。
2. `reflection.js` を器化：左メイン（ビュー切替）＋右サイドバー。配分は独立ビュー化、簡略リボン撤去、右レール撤去。
3. 日付ストリップ（気分ドット）を実装し `activeDate` を単一所有化。
4. 明日の計画ビュー（候補チップ＋クイック入力＋縦リスト＋並べ替え）＋ `createTask(due=翌日)` 連携。
5. `index.html`/`main.js` からタイムラインタブ除去、`#timeline` を振り返りへ移行。
6. `ref/reflection/` 更新＋スクショ検証、デモ追従、e2e/デモ確認。
- ロールバック: 表示レイヤー中心の変更のためコミット単位で revert 可能。データ・API・集計は不変なので後方互換。

## Open Questions

- 明日の計画の「分岐（課題あれば…）」は当面 **1タスクのテキスト**で残す方針でよいか（実行時評価はしない）。
- 左右比率（右サイドバー幅）とビュー数（将来「目標の日記」等を左ビューへ足す余地）は実装時の微調整とする。
- 候補チップ抽出のヒューリスティック閾値（分割単位・最大候補数）は実装時に決める。
