## Why

振り返りは「今日の作業記録（タイムライン）を見ながら書き、書き終えたら明日のタスクを登録する」一連のリチュアルなのに、現状は **タイムラインタブ** と **振り返りタブ** が分断され、振り返りタブ内のタイムラインは簡略リボンの簡易表示に留まる。さらに明日のタスク登録は「明日の計画へ →」でカンバンタブへ画面遷移するため、書いた段取り本文を見失いながらタブを往復する羽目になる（issue #61）。「見ながら書く」「見ながら登録する」が両方とも成立していない。

## What Changes

- **タイムラインタブを振り返りタブへ統合する**（**BREAKING**: 独立した「タイムライン」タブを廃止し、ナビゲーションは `今日 / 振り返り / カンバン / 目標 / 設定` の5タブになる）。統合後のタブ名は既存の呼び名どおり **「振り返り」** のまま。
- **左右2分割ワークスペースにする**。左＝大きなメイン、右＝細い振り返りサイドバー（常時表示）。左の完全版タイムラインを見ながら、右で振り返り本文を書ける。
- 左メインを **Notion 風のビュー切替（横スクロール・スナップ）** にし、`タイムライン（完全版・一日の作業記録） / 一日の配分 / 明日の計画` の3ビューをめくる。
- **一日の配分をエディタから分離**し、左メインの独立した1ビューにする（従来はエディタ上部の常設バー）。
- **右オーバーレイの簡略リボン・タイムラインを廃止**する（左メインの完全版タイムラインが役割を引き継ぐ）。
- 対象日ナビゲーションを **日付ストリップ**（前後日を気分ドット付きで並べ、‹ › とチップタップと「今日」で移動）へ刷新し、画面全体（全ビュー＋サイドバー）の対象日を駆動する。従来の右レール「過去の振り返り一覧」はこのストリップへ役割統合する。
- **明日の計画ビューを新設**する。振り返り本文に書いた明日の段取りを見ながら、本文から拾ったチップのタップ or 大きな入力欄への直接入力で、**一個ずつ**タスクを登録する。登録先は本物のカンバン（`due = 明日` の未完了タスク）で、別データストアは作らない。従来の「明日の計画へ → カンバンタブへ遷移」動線を置き換える。
- 受け入れ基準は companion `ユーザーフロー.md` を唯一の正典とし、視覚要件は参照実装 `ref/` へ写経してスクショ検証する（別物事故の再発防止）。

## Capabilities

### New Capabilities
- `reflection-timeline-workspace`: 統合後の振り返りタブの器。左＝メイン（ビュー切替の横スクロール・デッキ：タイムライン/一日の配分/明日の計画）＋右＝常時表示の振り返りサイドバー、上部の日付ストリップ（対象日を全体へ駆動）、タイムラインタブ廃止に伴うナビゲーション/ディープリンクの移行を規定する。
- `tomorrow-plan-capture`: 明日の計画ビュー。本文から拾ったチップまたは直接入力で、明日（`due=翌日`）の未完了タスクを本物のカンバンへ一個ずつ登録・並べ替えする動線を規定する。

### Modified Capabilities
- `reflection-journal`: 本文エディタと気分入力を右サイドバーへ再配置する。右レールの「過去の振り返り一覧」を日付ストリップへ役割移管する（このタブでの一覧UIは廃止）。視覚要件の参照先を新しい `ref/` の統合ワークスペースへ更新する。本文の意味（`content.trim().length>0` を「振り返り完了」とみなす PLANNING シグナル）・保存動線・エディタ挙動は変更しない。
- `reflection-day-overview`: 「一日の配分」を左メインの独立した全幅ビューへ昇格する（エディタ上部常設バーではなくなる）。右オーバーレイのグラフィカル縦帯タイムライン（簡略リボン）を廃止する。配分の集計規則（持ち分・名前＋色 identity・端〜端の母数・未記録）は変更しない。

## Impact

- **フロント**: `server/static/index.html`（タブ定義から「タイムライン」を除去）、`server/static/js/main.js`（ルーティング・`#timeline` ディープリンクの移行）、`server/static/js/reflection.js`（左右2分割・ビュー切替の器化）、`server/static/js/timeline.js`（描画を振り返りワークスペース内のビューとしてホスト）、`server/static/js/kanban.js`（`setTomorrowMode` 連携／明日タスクのクイック登録API利用）、`server/static/css/app.css`（新レイアウト `rf-*` クラス群）。
- **参照実装**: `ref/reflection/` に統合ワークスペースの HTML を写経（既存 `ref/timeline/`・`ref/reflection/` を統合参照へ更新）。
- **API**: 既存の read API（`getTimeline` / `getAllocation` / `getReflection(s)`）と task API（`createTask` due 指定）を再利用。新規サーバAPIは原則不要（明日タスク登録は既存 `createTask` を使う）。
- **データ/集計**: タイムライン・配分の集計経路、`reflection_entry` の意味、カンバンの `due`/`tomorrow` 判定は不変。デモモード（`demo-seed.ts`）の閲覧表示は新レイアウトへ追従が必要。
- **非対象**: タイムラインのラン結合・離席・同時記録などの描画規則（`timeline-run-view` 他）そのものは変更しない。カンバンの列・並べ替え・詳細エディタの既存仕様も変更しない。
