# ユーザーフロー（long-term-goal-loop / issue #54）

実際の画面（タブ）を歩いて、長期目標のループを1周する。
**題材**：目標「髪質を改善する」で、ボリュームアップシャンプーに賭けて、写真と質問で答え合わせする。

タブ構成（現状）：`今日` / `タイムライン` / `カンバン` / `振り返り` / `目標` / `設定`

## 骨格

この app の本質は **PDCA ループ**。それを比喩ではなく画面にそのまま出す。

```
Plan   賭け（仮説・計画・方針変更）を短文で書く          → 沿革に載る
Check  その答え合わせ。1つの Plan に複数ぶら下げられる    → 沿革に載る
日記   自由記入。今までどおり                           → 沿革に載らない（④で参照）
```

**沿革に載る／載らないの線引きは「大きさ」ではなく「検証がぶら下がっているか」で決まる。**

画面の分業（既存の app の筋に合わせる）：

```
振り返りタブ … 書く（Plan を立てる・Check を仕掛ける・日記）
今日タブ    … 詰まる（Check がゲートを閉じる・その場で答える・やめる）
目標タブ    … 読む（レポート＝目標の家。走行中も開ける・⑤沿革）
```

❓ は「まだ決まっていない・要判断」の印。

## 長期目標を作成するとき

*（既存フロー。変更なし。文脈のため記載）*

1. 目標タブ：「＋ 目標を作成」を押す
2. 目標タブ：名前「髪質を改善する」・目的の一文・開始日（今日から／明日から）を入れる
3. 目標タブ：実践を採用する（既存ルール条件から選ぶ／その場で新規作成）
4. 目標タブ：（任意）初日の写真をステージングする
5. 目標タブ：作成 → 一覧に「髪質を改善する [Day 1/30]」カードが現れる

## Plan を立てて、Check を仕掛けるとき

*（★#54 の中心。1つのフォームで完結する）*

1. 振り返りタブ：いつもどおり今日の振り返りを書く
2. 振り返りタブ：目標「髪質を改善する」のコーナーに **「＋ Plan」** ボタン。押す
3. 振り返りタブ：**Plan** を短文で書く
   > 「ボリュームアップシャンプーを使えば髪質が良くなるのではないだろうか」

   *購入も根拠もこの一文に溶ける。「買った」という種別は要らない。*
4. 振り返りタブ：**「＋ Check」** で答え合わせを足す（複数可）

   Check のフォームは **独立した2軸**でできている：

   ```
   ┌─────────────────────────────────────────────┐
   │ 種類  ● 📷 写真を投稿する   ○ 💬 質問に答える  │ ← 軸1
   │        撮るもの「前髪・正面」                  │
   │        （質問なら質問文を先に書く）             │
   ├─────────────────────────────────────────────┤
   │ いつ  [ 単発 ⇄ 範囲 ] ← トグル                │ ← 軸2（種類とは無関係）
   │        単発 … 3日後（or 7/18 と直接指定）      │
   │        範囲 … 3日後から 7日間（毎日）          │
   ├─────────────────────────────────────────────┤
   │ 場所メモ「洗面所」 時刻メモ「朝」（任意）        │
   │   ※ どちらも判定には使わない                   │
   └─────────────────────────────────────────────┘
   ```

   **★2軸は独立している。** 種類が「いつ」を決めることはない：
   - 📷 写真 × **範囲** … 7日間、毎日 髪の写真を撮って変化を追う
   - 💬 質問 × **単発** … 7/18 の一度だけ「使用感はどうだった？」に答える
   - どの組み合わせも作れる

   この例では2つ足す：
   - **Check①**：📷 写真「前髪・正面」／ **単発**・3日後（＝7/18）
   - **Check②**：💬 質問「ボリュームアップシャンプーの使用感はどうだった？」／ **範囲**・3日後から7日間

5. 振り返りタブ：その下の **日記**（既存の自由記入エディタ）は今までどおり書く
6. 振り返りタブ：保存する
7. 振り返りタブ：コーナーに「仕掛け中：📷前髪・正面（7/18）／💬使用感（7/18〜7/24）」が並ぶ

> この時点で沿革に Plan が1件積まれる。**7/18 まではゲートに何の影響もない**（Check は休眠中）。

**Check を1つも付けない Plan も作れる。** 方針だけ書きたいとき（例：「ブログはやめる。反応が薄いから」）は
Plan 単体で保存でき、沿革にはそのまま載る。

## Check の日が来て、気づくとき

*（7/18 当日。★ここが機能するかどうかの生命線）*

1. **その日はじめてダッシュボードを開いた瞬間、トーストが1回出る**
   > 📷 前髪・正面 ― 今日の Check です（髪質を改善する）

   *ゲートで詰まれば自ずと気づくが、不足条件は一覧の一行でしかない。
   初回オープンでトーストを出せば、**一日終わりかけて気づく**事故を減らせる。
   同じ日に2回目以降は出さない（うるさくしない）。*
2. 今日タブ：開く。作業時間・内訳を見る
3. 今日タブ：**ゲートの不足条件に Check が並ぶ**
   ```
   総作業時間 4h                        ✅
   翌日の計画                            ✅
   📷 前髪・正面                         ❌ 写真がまだ
       └ Plan: ボリュームアップシャンプーを使えば…
   💬 使用感はどうだった？               ❌ 未回答（7/18〜7/24 の1日目）
   ```
4. 今日タブ：他の条件を全部満たしても、**パスワードは表示されない**
5. 今日タブ：不足条件の行から直接 **答えられる**

> **通知の実装方針**：**時刻でスケジュールしない**（croner も OS トーストも使わない）。
> ダッシュボードをその日はじめて開いたとき、既存の `toast()` で1回出すだけ。
> 「その日もう出したか」は day_key 単位のフラグで判定する。
> OS 常駐トースト（pwsh / WinRT）は不要 ― 実装は軽く、完全ローカル・オフライン原則もそのまま。

## Check に答えるとき

### 📷 写真の Check

1. 今日タブ：不足条件の「📷 前髪・正面」から **「写真を出す」**
2. 今日タブ：画像を貼る／ファイルを選ぶ（キャプションは**先に決めてあるので入力不要**。確認だけ）
3. 今日タブ：提出 → 条件が ✅ に変わる
4. 今日タブ：他の条件も満たしていれば**パスワードが表示される**

> 範囲Check（例：7日間毎日写真）なら、毎日この提出を繰り返す。
> 同じキャプションなので③Before/After の同じ列に時系列で積み上がる。

> 写真は目標の 7/18 に保存され、キャプション「前髪・正面」で
> レポート③ Before/After 列に自動で入り、⑤沿革にも載る。

### 💬 質問の Check

1. 今日タブ：不足条件の「💬 使用感はどうだった？」から **「答える」**
2. 今日タブ：質問文が出ているので、答えを書く
   > 「泡立ちは良い。乾燥は減った気がする」
3. 今日タブ：保存 → 条件が ✅ に変わる

> 答えは⑤沿革に **Q&A のペア**で載る。これが「工夫→結果」の記録そのもの。

❓ **今日タブから直接答える**前提で書いた（＝ブロックされている場所で解決できる）。
　「書くのは振り返り」の筋からは外れるが、ゲート前で足止めして別タブへ飛ばすのは酷なので。これでいい？

## Check をサボった日／範囲Check の扱い

単発と範囲で**意図が違う**ので、サボりの扱いも分ける：

```
📌 単発Check（ピンポイント1日）
    出すまでロックを繰り越す。7/18 に出さなければ 7/19 も 7/20 も開かない。
    → 「遅れてでも出す価値がある」ため

📅 範囲Check（毎日 n 日間）
    その日限りのゲート。7/20 にサボれば 7/20 は開かない。
    7/21 は 7/21 の分だけを要求する（7/20 の分は繰り越さない）。
    範囲を過ぎたら消える。
    → 「7/20 の髪の写真」は 7/21 には撮れない（代替不能）ため
    → 沿革には「7日中5日提出」と事実どおり残る
```

## Check や Plan をやめるとき（理由つき）

*（唯一の脱出弁。逃げた事実そのものが歴史に残る）*

1. 今日タブ：不足条件の「📷 前髪・正面」の行から **「やめる」**
2. 今日タブ：**理由の入力を求められる（必須・空は拒否）**
   > 「シャンプーが肌に合わず返品した」
3. 今日タブ：実行 → その Check がゲートから外れる
4. 今日タブ：他の条件を満たしていればパスワードが表示される

Plan ごと取り下げる場合：

1. 振り返りタブ：目標コーナーの Plan から **「取り下げる」**＋理由
2. ぶら下がる Check がすべて外れる
3. **Plan は沿革に残る**（「取り下げた ― 理由」つき）。消さない

## 走行中に、いまの姿を眺めるとき

*（★「30日待たないと姿が分からない」を解消する。新画面は作らず、既存レポートの鍵を外す）*

1. 目標タブ：進行中カード「髪質を改善する [Day 12/30]」に **「レポートプレビュー」** ボタン
2. レポート：ヘッダに **目的の一文**・期間・Day 12/30・現時点の達成日数
3. レポート ①達成カレンダー：
   - Day 1〜12 は事実どおり（達成／未達成）
   - **Day 13〜30 は「未到来」として空白** ← ★未達成の黒星で埋めない
4. レポート ②時間の推移：Day 12 までの折れ線
5. レポート ③Before/After：今ある写真で（Before＝初日、After＝いちばん新しい）
6. レポート ④日記リーダー：カレンダーのマスを押すとその日の日記本文
7. レポート **⑤沿革**：Plan と Check の答え合わせが時系列に並ぶ
8. レポート：**最終日写真のCTAは出さない**（完走後だけの導線）

> ボタンの文言は状態で分ける：進行中は **「レポートプレビュー」**、完走後は既存の **「レポートを開く」**。
> 同じ画面だが、走行中はまだ途中の姿であることが一目で伝わる。

## 1ヶ月後、沿革を読むとき

1. 目標タブ：完走カード「髪質を改善する [完走]」→「レポートを開く」
2. レポート：ヘッダ＋①〜④（既存どおり）
3. レポート **⑤沿革**：1ヶ月の「賭けと答え合わせ」が時系列で並ぶ
   ```
   7/15  Plan  ボリュームアップシャンプーを使えば髪質が良くなるのでは
           ├ 📷 7/18  前髪・正面                          [画像]
           └ 💬 7/18〜7/24  使用感はどうだった？（7日中5日）
                 7/18 泡立ちは良い。乾燥は減った気がする
                 7/19 …

   7/22  Plan  夜シャンに変える。朝だと乾かす時間がない
           └ 📷 7/25  前髪・正面                          [画像]

   7/29  Plan  頭皮マッサージを毎日やる
           └ やめた ― 続かなかった。3日で飽きた
   ```
4. レポート：最終日写真のCTAから、完走後でも Day30 の写真を足せる（既存）

## 今回やらないこと

- 追跡グループの入替／作業時間の器／折れ線の優雅な畳み込み（#54 の D スレッド）
- 撤退条件のシステム化（「できなければインターン」）
- 沿革の縦書き・ミニマル組版、LLM によるテキスト整形（**完全ローカル・オフライン原則を維持**）
- 場所・時刻の機械検証（app は位置情報を持たない。名誉ベースのメモに留める）
- ルールロック概念（日次実践のロック／アンロック）
- Plan の種別選択（仮説／方針変更／計画）。Plan の文を読めば分かるので今回は付けない

## 決まったこと

- **Plan / Check** の文言で通す（種別選択は付けない）
- Check の **種類（📷写真／💬質問）と いつ（単発／範囲）は独立した2軸**。トグルで自由に指定
- **単発Check** ＝ 出すまでロック繰り越し。**範囲Check** ＝ その日限り・繰り越さない・期間を過ぎたら消える
- **Check なしの Plan** も作れる（方針だけ書く）
- 写真提出・質問回答は **今日タブから直接**やる
- **通知を入れる**。時刻指定ではなく、**その日はじめてダッシュボードを開いたとき1回**（アプリ内トースト）
- 進行中のボタンは **「レポートプレビュー」**、完走後は「レポートを開く」
- 日記は沿革に載せない。④日記リーダーで参照する
